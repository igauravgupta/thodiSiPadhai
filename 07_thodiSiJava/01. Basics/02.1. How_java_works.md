# What JDK, JRE, and JVM are — and what they include

## 1. JVM (Java Virtual Machine)

- The engine that runs Java bytecode.
- Executes `.class` files.
- Part of both the JRE and the JDK.

## 2. JRE (Java Runtime Environment)

- Definition: `JRE = JVM + core libraries + runtime tools`
- Purpose: Used to run Java programs (not to develop them).

Includes:

- JVM
- Java standard libraries (`java.*`, `javax.*`, etc.)
- Supporting files (properties, security files)

Does not include:

- Compiler (`javac`)
- Development tools

## 3. JDK (Java Development Kit)

- Definition: `JDK = JRE + development tools`
- Purpose: Used to develop and run Java programs.

Includes:

- Everything in the JRE (JVM + core libraries)
- Compiler (`javac`)
- Debugger (`jdb`)
- Javadoc generator (`javadoc`)
- Packaging tools (`jar`)
- Other development utilities

## Do modern JDKs include a JRE folder?

- Modern JDKs (Java 11 and later):
  - Do **not** include a separate `jre/` folder, but they contain everything a JRE provides.
  - JVM is always included.
- Older JDKs (Java 8 and earlier):
  - Included a separate `jre/` folder inside the JDK directory.

## Summary

| Component | Contains               | Purpose            |
| --------- | ---------------------- | ------------------ |
| JVM       | Executes bytecode      | Run Java           |
| JRE       | JVM + libraries        | Run Java apps      |
| JDK       | JRE + compiler + tools | Develop & run Java |
