# Downcasting in Java

## Question: Can We Do This?

Given:

- `class A` → Parent class
- `class B` → Child class

**Is this possible?**

```java
B obj = new A();
```

### Short Answer: ❌ **NO**

This will **NOT compile** and gives a compiler error.

---

## Why Is This Not Allowed?

### Understanding the IS-A Relationship

In Java, inheritance follows the **IS-A** relationship:

- ✅ Every `B` IS-A `A` (Child is a Parent) → **Valid**
- ❌ Not every `A` IS-A `B` (Parent is not necessarily a Child) → **Invalid**

### What's Allowed vs Not Allowed

#### ✅ Upcasting (Child → Parent) - Allowed

```java
A obj = new B();   // OK - Implicit casting
```

This works because a `B` object has all the features of `A`.

#### ❌ Downcasting Without Proper Object - Not Allowed

```java
B obj = new A();   // Compiler Error
```

**Compiler Error:**

```
Type mismatch: cannot convert from A to B
```

---

## What Happens If You Force Cast?

### Scenario 1: Explicit Cast

```java
B obj = (B) new A();   // ⚠️ Compiles, but...
```

This will **compile** but throws a **runtime exception**:

```
java.lang.ClassCastException: class A cannot be cast to class B
```

**Why?** Because the actual object is of type `A`, and an `A` object cannot behave like a `B` object.

---

## When Is Downcasting Safe?

### ✅ Safe Downcasting Example

Downcasting is **only safe** when the actual object is of the child type:

```java
// Step 1: Upcast - Child object stored in parent reference
A obj = new B();   // OK - Upcasting

// Step 2: Downcast - Safe because actual object is B
B child = (B) obj;   // OK - Safe downcasting
```

This works because the actual runtime object is `B`, not `A`.

### ❌ Unsafe Downcasting Example

```java
A obj = new A();      // Actual object is A
B child = (B) obj;    // ❌ ClassCastException at runtime
```

This fails because you're trying to cast a parent object to a child type.

---

## Key Takeaways

| Operation                | Syntax                                     | Compile Time | Runtime      | Status             |
| ------------------------ | ------------------------------------------ | ------------ | ------------ | ------------------ |
| **Upcasting**            | `A obj = new B();`                         | ✅ OK        | ✅ OK        | Always Safe        |
| **Downcasting (Wrong)**  | `B obj = new A();`                         | ❌ Error     | -            | Not Allowed        |
| **Downcasting (Forced)** | `B obj = (B) new A();`                     | ✅ OK        | ❌ Exception | Compiles but Fails |
| **Downcasting (Safe)**   | `A obj = new B();`<br>`B child = (B) obj;` | ✅ OK        | ✅ OK        | Safe               |

---

## Summary

> **You cannot assign a parent object to a child reference because a parent object does not have all the features of a child.**

- **Upcasting** (Child → Parent): Always safe and automatic
- **Downcasting** (Parent → Child): Only safe when the actual object is of the child type

### Best Practice

Use `instanceof` to check before downcasting:

```java
A obj = new B();

if (obj instanceof B) {
    B child = (B) obj;  // Safe
    // Use child object
}
```
