    # IntelliJ IDEA — Setup (Linux)

    ## Prerequisites
    - Downloaded IntelliJ IDEA tar.gz is in ~/Downloads.
    - You have sudo privileges.

    ## 1. Move IntelliJ to /opt (optional, recommended)
    Move the extracted folder to a stable location:

    ```bash
    sudo mv ~/Downloads/idea-* /opt/intellij
    sudo chown -R root:root /opt/intellij   # optional: set ownership
    ```

    Now IntelliJ is at: `/opt/intellij`.

    ## 2. Run IntelliJ for the first time
    Start the IDE from its bin folder:

    ```bash
    cd /opt/intellij/bin
    ./idea.sh
    ```

    This launches IntelliJ and performs initial setup.

    ## 3. Add a convenient launcher (optional)
    Option A — add to PATH (works for bash/zsh):

    ```bash
    # Bash
    echo 'export PATH="$PATH:/opt/intellij/bin"' >> ~/.bashrc
    source ~/.bashrc

    # Zsh
    echo 'export PATH="$PATH:/opt/intellij/bin"' >> ~/.zshrc
    source ~/.zshrc
    ```

    Now you can run the IDE with:

    ```bash
    idea
    ```

    Option B — create a system symlink:

    ```bash
    sudo ln -s /opt/intellij/bin/idea.sh /usr/local/bin/idea
    ```

    ## 4. Create a Desktop Entry (recommended)
    Start IntelliJ once, then use the menu:

    Tools → Create Desktop Entry…

    Check:
    - Add to the system menu
    - Create a desktop shortcut (if available)

    Click OK.

    (Optional) Manually create a .desktop file:

    ```ini
    [Desktop Entry]
    Version=1.0
    Type=Application
    Name=IntelliJ IDEA
    Exec=/opt/intellij/bin/idea.sh %f
    Icon=/opt/intellij/bin/idea.png
    Categories=Development;IDE;
    Terminal=false
    ```

    Save as `/usr/share/applications/jetbrains-idea.desktop` (requires sudo).

    ## 5. Notes & Troubleshooting
    - If the `idea` command doesn't start, verify the PATH or symlink.
    - Ensure the `idea.sh` is executable: `sudo chmod +x /opt/intellij/bin/idea.sh`.
    - To remove, delete `/opt/intellij` and any created symlink or desktop file.

    That's it — IntelliJ should now be easy to launch from terminal or your desktop environment.
