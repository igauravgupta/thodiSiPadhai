# useCallback

## What it is
`useCallback` is a React Hook that returns a memoized function. It preserves the same function reference across renders unless one of its dependencies changes. This helps avoid unnecessary re-renders or re-effects when passing callbacks to child components or using them in dependency arrays.

Syntax:
```js
const memoizedCallback = useCallback(() => {
    // function body
}, [dep1, dep2]);
```

## When to use (use cases)
- Passing callbacks to memoized child components (React.memo) to avoid triggering re-renders due to changing function references.
- Using a callback inside another hook (e.g., `useEffect`, `useMemo`) where referential equality matters.
- Preventing re-creation of expensive callback closures when dependencies rarely change.

Don’t use it everywhere — it adds complexity and a small runtime cost. Prefer it when you have an actual performance or referential-equality problem.

## Example

Parent passes an `onClick` handler to a memoized child. Without `useCallback`, the child would re-render every time because the handler reference changes.

```jsx
import React, { useState, useCallback } from "react";

const Child = React.memo(({ onClick, label }) => {
    console.log("Child render:", label);
    return <button onClick={onClick}>{label}</button>;
});

export default function App() {
    const [count, setCount] = useState(0);
    const [other, setOther] = useState(0);

    // Without useCallback, this function is recreated on every render
    // With useCallback it stays the same unless 'count' changes
    const increment = useCallback(() => {
        setCount(c => c + 1); // functional update avoids needing 'count' in deps
    }, []); // stable reference across renders

    const incrOther = () => setOther(o => o + 1);

    return (
        <div>
            <p>count: {count}</p>
            <p>other: {other}</p>
            <Child onClick={increment} label="Increment count" />
            <Child onClick={incrOther} label="Increment other" />
        </div>
    );
}
```

In this example:
- `Child` is wrapped with `React.memo`, so it only re-renders when props change.
- `increment` is stable because of `useCallback`; clicking the "Increment other" button will not cause the "Increment count" child to re-render.
- `incrOther` is recreated each render, so the corresponding child will re-render when parent renders.

## Caveats & tips
- `useCallback` memoizes the function reference, not the result. For memoizing computed values use `useMemo`.
- Keep dependency arrays correct. Missing dependencies cause stale closures; over-including makes the callback change more often.
- Sometimes using functional state updates (setState(prev => ...)) reduces dependencies.
- Measure before optimizing — many components don’t need it.
