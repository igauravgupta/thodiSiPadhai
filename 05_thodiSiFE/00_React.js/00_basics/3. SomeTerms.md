# React.js — Essential Terms & Concepts

## npx — Node Package eXecute

### What is npx?

npx is a command-line tool bundled with npm (since npm v5.2+) that runs Node packages without requiring a global install. It is useful for invoking CLI tools on-demand.

### How it works

- Downloads the requested package (if not available locally).
- Executes the package command.
- Does not persist the package globally (unless you explicitly install it).

### When to use

- Running a package one time (scaffolding, code generators, etc.).
- Trying a package without committing to a global install.
- Ensuring you run the version shipped with a project (via node_modules/.bin) or a specific remote version.

### Examples

Create a new React app with create-react-app:

```bash
npx create-react-app my-app
```

Run a package available in a project’s devDependencies:

```bash
npx eslint src/
```

### npm vs npx

| Feature               |                                 npm |                                    npx |
| --------------------- | ----------------------------------: | -------------------------------------: |
| Full form             |                Node Package Manager |                   Node Package Execute |
| Purpose               |             Install/manage packages |                       Execute packages |
| Installation behavior | Installs packages (global or local) |             Runs a package temporarily |
| Typical use case      |      Reuse packages across sessions | One-off executions (scaffolding, CLIs) |
| Example               |                   npm install react |            npx create-react-app my-app |

## JSX

### What is JSX?

JSX (JavaScript XML) is a syntax extension that lets you write HTML-like markup inside JavaScript. It is syntactic sugar that is compiled to React.createElement calls (or equivalent runtime calls) by tools like Babel.

JSX is transpiled (typically by Babel) into plain JavaScript that creates React elements, which React then renders into HTML (the DOM).

Use case — Combining markup and logic

Rules -

- Write JavaScript expressions inside JSX using curly braces: { expression }.
- Wrap multi-line JSX (or multi-line returns) in parentheses: ( ... ).
- Do not use the HTML attribute "class"; use "className" in JSX.

Render as -

- <ComponentName/>
- ComponentName()

### Key points

- Improves readability when building UI components.
- Must be transpiled before running in browsers (Babel, TypeScript, or a build tool).
- You can embed JavaScript expressions using curly braces: { expression }.
- Components using JSX typically return a single root element (or use fragments).

### Example

A simple functional component using JSX:

```jsx
import React from "react";

function Greeting({ name }) {
  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>Welcome to React.</p>
    </div>
  );
}

export default Greeting;
```

---

Notes

- For modern React (17+), JSX transformation may not require explicit React imports depending on your build setup.
- Use npx for quick, one-off CLI tasks; install with npm when you need persistent access.
