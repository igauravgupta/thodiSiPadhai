# Routing in React.js

## Static Routing

React Router handles client-side routing by mapping URLs to React components. Typical setup lives near the app entry.

Key pieces:

- BrowserRouter — high-level router using the History API.
- Routes and Route — declare route-to-component mappings.
- Link / NavLink — navigation elements that avoid full-page reloads.
- Navigate — programmatic redirects (replacement for Redirect).
- Outlet — where child routes render in nested layouts.

Basic example:

```jsx
import { BrowserRouter, Routes, Route, Link } from "react-router-dom";

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
```

## Route props and features

- path — route pattern (supports parameters like /users/:id).
- element — component to render (JSX).
- index — marks an index (default) child route.
- path="\*" — catch-all for 404 pages.
- nested routes — define Route children; render with <Outlet/>.

Nested example:

```jsx
<Routes>
  <Route path="/dashboard" element={<DashboardLayout />}>
    <Route index element={<Overview />} />
    <Route path="settings" element={<Settings />} />
  </Route>
</Routes>;

function DashboardLayout() {
  return (
    <div>
      <Sidebar />
      <Outlet /> {/* nested routes render here */}
    </div>
  );
}
```

## Links and active state

- Link prevents page reloads; NavLink provides active styling.

```jsx
<NavLink to="/about" className={({ isActive }) => (isActive ? "active" : "")}>
  About
</NavLink>
```

## Dynamic Routing

- useParams() — read route params (e.g., id from /users/:id).
- useLocation() — current location object (pathname, search, state).
- useSearchParams() — read/update URL query string easily.
- useNavigate() — programmatic navigation (push or replace).

Examples:

```jsx
// Read path param
import { useParams } from "react-router-dom";
function User() {
  const { id } = useParams();
  // fetch user by id...
}

// Navigate programmatically
import { useNavigate } from "react-router-dom";
const nav = useNavigate();
nav("/login"); // push
nav("/home", { replace: true }); // replace
```

Read query params:

```jsx
import { useSearchParams } from "react-router-dom";
const [searchParams] = useSearchParams();
const page = searchParams.get("page");
```

Passing state:

```jsx
navigate("/details", { state: { from: "homepage" } });
// read via useLocation().state
```

## Redirects and protected routes

- Use <Navigate to="/login" replace /> for redirects.
- Protect routes by wrapping them in a guard component that checks auth and returns either the child <Outlet/> or a <Navigate/>.

Protected route example:

```jsx
function PrivateRoute({ isAuth }) {
  return isAuth ? <Outlet /> : <Navigate to="/login" replace />;
}

// usage
<Route element={<PrivateRoute isAuth={user != null} />}>
  <Route path="/profile" element={<Profile />} />
</Route>;
```

## Code-splitting routes

Use React.lazy + Suspense for lazy route components:

```jsx
const Settings = React.lazy(() => import("./Settings"));

<Route
  path="/settings"
  element={
    <Suspense fallback={<Loading />}>
      <Settings />
    </Suspense>
  }
/>;
```

## Tips & best practices

- Keep routing configuration close to app root.
- Use nested routes and layout components for shared UI.
- Prefer useSearchParams for query strings instead of manual URL parsing.
- Use NavLink for active link styling.
- Handle unknown routes with a catch-all "\*" to show a 404 page.
- For SEO on client-side apps, configure server to serve index.html for app routes (or use SSR).
- Use basename on BrowserRouter if the app is served from a subpath: <BrowserRouter basename="/app">.

## Common pitfalls

- Forgetting <BrowserRouter> around Routes.
- Using anchor tags (<a>) instead of <Link> causing full reload.
- Heavy synchronous code in route components blocking rendering — use lazy loading.

This should give a compact but practical reference for routing with React Router.
