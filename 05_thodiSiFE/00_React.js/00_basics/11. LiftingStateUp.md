# Lifting State Up

"Lifting state up" is a React pattern where you move state that multiple components need to access or update to their nearest common ancestor. That ancestor becomes the single source of truth and passes data down via props and update functions down to children.

Why and when

- Synchronize data between siblings or components at different branches.
- Keep a single source of truth to avoid inconsistent UI.
- Prefer this for a small to moderate component tree; use Context or a state library if many levels or many consumers.

How to do it

1. Identify the state shared by multiple components.
2. Move that state into the nearest common parent using useState (or similar).
3. Pass the state value to children as props.
4. Pass setter/callback functions to children so they can request updates.

Example

```jsx
// parent
function Parent() {
  const [text, setText] = useState("");
  return (
    <>
      <InputA value={text} onChange={setText} />
      <InputB value={text} onChange={setText} />
    </>
  );
}

// child
function InputA({ value, onChange }) {
  return <input value={value} onChange={(e) => onChange(e.target.value)} />;
}
```

Tips

- Lift state only as high as necessary.
- Derive values from state when possible (avoid duplicating state).
- If many components need the same state deeply, consider Context or external state management.
- Keep update handlers simple and predictable.
