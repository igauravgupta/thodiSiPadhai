# Shimmer / Skeleton + useEffect

Two common loading patterns

1. App loads → API call → render fetched data (no intermediate UI)
2. App loads → render skeleton/loading UI → API call → render fetched data (recommended for perceived performance)

The second pattern is implemented with the useEffect hook and conditional rendering:

- Show a skeleton or spinner while the data is being fetched.
- Replace the skeleton with actual UI when fetch completes.

Example (React functional component)

```jsx
import { useState, useEffect } from "react";

function MyComponent() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    let mounted = true; // avoid state updates after unmount
    fetch("/api/data")
      .then((res) => res.json())
      .then((json) => {
        if (!mounted) return;
        setData(json);
        setLoading(false);
      })
      .catch(() => {
        if (!mounted) return;
        setLoading(false);
      });
    return () => {
      mounted = false;
    };
  }, []);

  return loading ? <SkeletonUI /> : <DataView data={data} />;
}
```

What conditional rendering means

- Render different JSX based on state (loading, error, empty, success).
- Typical branches: loading skeleton, error message, empty state, data view.

useEffect dependency array — three cases

1. [dep] (one or more dependencies): effect runs after initial render and again whenever any dep changes.
2. [] (empty array): effect runs once after initial render (componentDidMount behavior).
3. no array: effect runs after every render (not common for data fetching; can cause infinite loops if it updates state).

Practical tips

- Use a loading boolean or check data === null to decide whether to show skeleton.
- Include cleanup inside useEffect to avoid setting state on unmounted components (abort controller or mounted flag).
- Be careful with objects/functions in the dependency array — use useCallback/useMemo or list specific primitive dependencies to avoid unnecessary re-runs.
- Handle errors and show an error UI if the fetch fails.
- Keep skeletons lightweight (pure CSS) for best performance.

Short checklist for data fetching UI

- Start loading state true.
- Fetch in useEffect.
- On success: set data, set loading false.
- On error: set loading false, set error state.
- Render: loading → skeleton, error → error UI, data → final UI.
- Clean up fetch or subscriptions on unmount.

## Cleanup in function components

- In function components useEffect can return a cleanup function. React runs it when the component unmounts or before the effect runs again (when dependencies change).

Examples

- Abort in-flight fetches (preferred over isMounted flag)
```jsx
useEffect(() => {
    const ctrl = new AbortController();
    fetch("/api/data", { signal: ctrl.signal })
        .then(r => r.json())
        .then(setData)
        .catch(err => {
            if (err.name !== "AbortError") setError(err);
        });
    return () => ctrl.abort(); // cancels fetch on unmount / deps change
}, []);
```

- Remove event listeners
```jsx
useEffect(() => {
    const onResize = () => setSize(window.innerWidth);
    window.addEventListener("resize", onResize);
    return () => window.removeEventListener("resize", onResize);
}, []);
```

- Clear timers / intervals
```jsx
useEffect(() => {
    const id = setInterval(tick, 1000);
    return () => clearInterval(id);
}, []);
```

- Unsubscribe from subscriptions / sockets
```jsx
useEffect(() => {
    const sub = chat.subscribe(msg => setMsgs(m => [...m, msg]));
    return () => sub.unsubscribe();
}, []);
```

Notes
- Prefer AbortController for fetch cancellation instead of setting a mounted flag.
