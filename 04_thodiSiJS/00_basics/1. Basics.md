## Ways to Link JS with HTML

1. **Inline Script**

   - Add JavaScript directly inside an HTML element using the `onclick`, `onchange`, etc. attributes.
   - Example:
     ```html
     <button onclick="alert('Hello!')">Click Me</button>
     ```

2. **Internal Script**

   - Place JavaScript code inside a `<script>` tag within the HTML file, usually in the `<head>` or at the end of the `<body>`.
   - Example:
     ```html
     <script>
       console.log("Hello from internal script!");
     </script>
     ```

3. **External Script**
   - Link to an external `.js` file using the `<script src="path/to/file.js"></script>` tag.
   - Example:
     ```html
     <script src="script.js"></script>
     ```

**Note:**

- Placing scripts at the end of the `<body>` ensures the HTML loads before the JS runs.
- Use the `defer` or `async` attributes for better loading performance with external scripts.

---

## What happens when both internal and external scripts are added?

- Scripts are executed in the order they appear in the HTML file.
- If an external script is placed before an internal script, the external script loads and runs first, then the internal script.
- If both are in the `<head>` without `defer` or `async`, the browser loads and executes them sequentially, blocking HTML parsing.
- Using `defer` on both ensures they execute in order after the HTML is parsed.
- If both scripts define the same variable or function, the last one loaded will overwrite the previous one.

---

## Difference Between `defer` and `async` in JavaScript

| Attribute | When Script Loads           | When Script Executes      | Order Guaranteed? |
| --------- | --------------------------- | ------------------------- | ----------------- |
| (none)    | As soon as browser finds it | Immediately (blocks HTML) | Yes (in order)    |
| defer     | As soon as browser finds it | After HTML parsed         | Yes (in order)    |
| async     | As soon as browser finds it | As soon as loaded         | No (out of order) |

- **defer**: Scripts are downloaded in parallel but executed in order after the HTML is parsed.
- **async**: Scripts are downloaded in parallel and executed as soon as they finish loading (order not guaranteed).
- **No attribute (normal)**: Script is loaded and executed immediately, blocking HTML parsing.

---

## Scenarios

### 1. One async, one defer

```html
<script src="a.js" async></script>
<script src="b.js" defer></script>
```

- `a.js` (async) may execute at any time after it loads (order not guaranteed).
- `b.js` (defer) will execute after HTML parsing is complete (order guaranteed among defer scripts).

### 2. Both async

```html
<script src="a.js" async></script>
<script src="b.js" async></script>
```

- Both scripts load in parallel and execute as soon as each finishes loading (order not guaranteed).

### 3. One async, one normal

```html
<script src="a.js" async></script>
<script src="b.js"></script>
```

- `a.js` (async) loads in parallel and executes as soon as it loads.
- `b.js` (normal) blocks HTML parsing and executes immediately when encountered.

### 4. One normal, one defer

```html
<script src="a.js"></script>
<script src="b.js" defer></script>
```

- `a.js` (normal) blocks HTML parsing and executes immediately.
- `b.js` (defer) executes after HTML parsing is complete.

## Static vs Dynamic Languages

- **Static Language:**

  - Type checking is done at compile time.
  - Variable types must be declared and cannot change.
  - Errors are often caught before running the program.
  - also known as strongly typed language.
  - Examples: Java, C, C++, TypeScript

- **Dynamic Language:**
  - Type checking is done at runtime.
  - Variable types can change as the program runs.
  - Errors may only appear when the code is executed.
  - also known as loosly typed language.
  - Examples: JavaScript, Python, Ruby, PHP

**Key Differences:**
| Feature | Static Language | Dynamic Language |
|-----------------|---------------------|----------------------|
| Type Checking | Compile time | Runtime |
| Type Declaration| Required | Not required |
| Flexibility | Less (fixed types) | More (types can change) |
| Error Detection | Before execution | During execution |

## Variables in JavaScript

- Variables store data values that can be used and changed in a program.
- Declared using `var`, `let`, or `const`.
- Variable names are case-sensitive and must start with a letter, $, or \_.

### var vs let

| Feature            | var                                | let                                    |
| ------------------ | ---------------------------------- | -------------------------------------- |
| Scope              | Function-scoped (Global)           | Block-scoped (Local)                   |
| Hoisting           | Hoisted (initialized as undefined) | Hoisted (not initialized)              |
| Redeclaration      | Can be redeclared in same scope    | Cannot be redeclared in same scope     |
| Reassignment       | Allowed                            | Allowed                                |
| Temporal Dead Zone | No                                 | Yes (cannot access before declaration) |

**Summary:**

- Use `let` for variables that may change and are block-scoped.
- Avoid `var` in modern JS; use `let` or `const` for predictable behavior.
- Const variable -
  1. Scope - Local
  2. Always provide value during initialization
- In production we prefer using `const` variable.

## Data Types in JavaScript

JavaScript has dynamic types, meaning variables can hold any type of value. The main data types are:

### 1. Primitive Types

- **String**: Textual data and can be declared using both '' and "". Example: `'hello'`, `"world"`
- **Number**: Numeric values (integer or floating point). Example: `42`, `3.14`
- **BigInt**: Large integers. Example: `123n`
- **Boolean**: Logical true/false. Example: `true`, `false`
- **Undefined**: Variable declared but not assigned a value. Example: `let x; // x is undefined`
- **Null**: Explicitly no value. Example: `let y = null;`
- **Symbol**: Unique and immutable value, often used as object keys. Example: `Symbol('id')`

### 2. Reference Types

- **Object**: Collection of key-value pairs. Example: `{ name: 'Alice', age: 25 }`
- **Array**: Ordered list of values. Example: `[1, 2, 3]`
- **Function**: Callable object. Example: `function greet() {}`
- **Date, RegExp, Map, Set, etc.**: Built-in object types for special purposes.

**Type Checking:**

- Use `typeof` to check most types: `typeof 42 // 'number'`
- Use `Array.isArray(arr)` to check for arrays.
- `typeof null` returns `'object'` (this is a historical quirk).

### Notes on `+` Operator

- If both operands are numbers, `+` adds them: `2 + 3 // 5`
- If either operand is a string, `+` concatenates: `'2' + 3 // '23'`, `2 + '3' // '23'`
- Use `Number()` or `parseInt()` to convert strings to numbers if needed.
- `console.log(true + 10); // 11` (true is converted to 1, so 1 + 10 = 11)
- `console.log('1' * 2); // 2` (the string '1' is converted to number 1, so 1 \* 2 = 2)
- `console.log('a' * 1); // NaN` (the string 'a' cannot be converted to a number, so result is NaN)
- `console.log('a' + 1) // 'a1'` (the number 1 is converted to a string and concatenated)

#### Tips

    1. String + String = String
    2. Number+ String = String
    3. String(Numberic string) * Number = Number

---

### Converting Number to String in JavaScript

- Use `String(num)`, `num.toString()`, or template literals `` `${num}` ``.
- Example: `String(42)`, `(42).toString()`, `` `${42}` `` all produce `'42'`.

## Guess the Number Game (do...while Example, with Exit)

```js
const secret = Math.floor(Math.random() * 10) + 1; // random number 1-10
let guess;
do {
  guess = Number(prompt("Guess a number between 1 and 10 (0 to quit):"));
  if (guess === 0) {
    alert("Game stopped.");
    break;
  }
  if (guess < secret) {
    alert("Too low!");
  } else if (guess > secret) {
    alert("Too high!");
  }
} while (guess !== secret);
if (guess === secret) {
  alert("Correct! The number was " + secret);
}
```

### Implications of Using `undefined` in JavaScript

- **`undefined`** indicates that a variable has been declared but not initialized.
- This is different from an empty value (`null` or `""`), which represents an intentional absence of value and may still occupy memory.
- Declaring a variable without assigning a value keeps it `undefined` until a value is provided.

### Distinction Between `undefined` and "not defined"

- **Not defined:** Refers to variables that have not been declared in any scope. Accessing such a variable results in a `ReferenceError: x is not defined`.
  ```js
  console.log(x); // ReferenceError: x is not defined
  ```
- **Undefined:** Refers to declared variables that have not been assigned a value. Accessing such a variable returns `undefined`.
  ```js
  let a;
  console.log(a); // undefined
  ```
  In summary, **"not defined"** means the variable doesn't exist, while **"undefined"** means the variable exists but has no value yet.

### Best Practices with `undefined`

JavaScript's loose typing allows variables to hold values of different data types throughout execution. However, directly assigning `undefined` to a variable is discouraged as it can lead to confusion and inconsistencies within the code. It is advisable to reserve `undefined` for its intended purpose—to indicate that a variable has not been initialized.

**Best Practices:**

- Avoid explicitly assigning `undefined` to variables.
- Use `null` to intentionally clear or reset a variable's value.
- Let JavaScript assign `undefined` automatically to uninitialized variables.
- Check for `undefined` when determining if a variable has been assigned a value.

By following these practices, your code remains more predictable and easier to maintain.
