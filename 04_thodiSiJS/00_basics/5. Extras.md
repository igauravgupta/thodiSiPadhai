# JavaScript Date Object

The `Date` object in JavaScript is used to work with dates and times.

## Creating Dates

```js
const now = new Date(); // current date and time
const specificDate = new Date("2024-06-01T10:00:00");
const fromComponents = new Date(2024, 5, 1, 10, 0, 0); // months are 0-indexed
```

## Common Methods

- `getFullYear()`: Gets the 4-digit year.
- `getDate()`: Gets the day of the month (1-31).
- `getDay()`: Gets the day of the week (0-6, Sun-Sat).
- `getHours()`, `getMinutes()`, `getSeconds()`: Get time components.
- `setFullYear()`, `setMonth()`, etc.: Set date/time components.
- `getTime()`: Gets milliseconds since Jan 1, 1970 (Unix Epoch).
- `toLocaleString()`: Returns a locale-sensitive string.

## Example

```js
const d = new Date();
console.log(d.getFullYear()); // e.g., 2024
console.log(d.toLocaleDateString()); // e.g., "6/1/2024"
```

## Notes

- Months are zero-based (January is 0).
- Useful for date calculations, formatting, and parsing.
- For advanced date manipulation, consider libraries like `date-fns` or `moment.js`.

## setTimeout and setInterval

JavaScript provides two important timing functions for scheduling code execution:

### `setTimeout`

Executes a function once after a specified delay (in milliseconds).

```js
setTimeout(() => {
  console.log("This runs after 2 seconds");
}, 2000);
```

### `setInterval`

Executes a function repeatedly, with a fixed time delay between each call.

```js
const intervalId = setInterval(() => {
  console.log("This runs every second");
}, 1000);

// To stop the interval:
clearInterval(intervalId);
```

**Note:** Use `clearTimeout` and `clearInterval` to cancel scheduled actions.

## JavaScript Currying

Currying is a technique in JavaScript where a function with multiple arguments is transformed into a sequence of functions, each taking a single argument.

### Example

```js
function add(a) {
  return function (b) {
    return a + b;
  };
}

const add5 = add(5);
console.log(add5(3)); // 8
```

### ES6 Arrow Function Syntax

```js
const multiply = (a) => (b) => a * b;
console.log(multiply(2)(4)); // 8
```

### Why Use Currying?

- Allows partial application of functions.
- Makes code more reusable and modular.
- Useful in functional programming patterns.

**Tip:** Libraries like Lodash provide utility functions for currying.

## Practical Currying Example: Email Sending

Currying can help pre-configure functions for specific use cases, such as sending emails with a preset sender.

```js
function sendEmail(from) {
  return function (to) {
    return function (subject) {
      return function (body) {
        // Simulate sending email
        console.log(`From: ${from}\nTo: ${to}\nSubject: ${subject}\n${body}`);
      };
    };
  };
}

const sendFromAdmin = sendEmail("admin@example.com");
const sendToUser = sendFromAdmin("user@example.com");
const sendWelcome = sendToUser("Welcome!");
sendWelcome("Hello and welcome to our service.");
```

This approach lets you create specialized email functions by partially applying arguments.

## Function Composition in JavaScript

Function composition is the process of combining two or more functions to produce a new function. The output of one function becomes the input of the next.

### Example

```js
const double = (x) => x * 2;
const square = (x) => x * x;

// Compose: square(double(x))
const compose = (f, g) => (x) => f(g(x));

const doubleThenSquare = compose(square, double);

console.log(doubleThenSquare(3)); // (3 * 2) ^ 2 = 36
```

**Tip:** Libraries like Lodash and Ramda provide utilities for composing multiple functions.

## Variadic Function Composition (Infinite Functions and Values)

You can create a function that accepts any number of functions and values, applying them in sequence.

### Example: Compose Many Functions

```js
function composeMany(...funcs) {
  return function (initialValue) {
    return funcs.reduceRight((acc, fn) => fn(acc), initialValue);
  };
}

const add1 = (x) => x + 1;
const times3 = (x) => x * 3;
const minus2 = (x) => x - 2;

const composed = composeMany(add1, times3, minus2);

console.log(composed(5)); // add1(times3(minus2(5))) => add1(times3(3)) => add1(9) => 10
```

### Example: Accepting Infinite Values

You can also write a function that takes any number of values and applies a composed function to each:

```js
function applyToAll(fn, ...values) {
  return values.map(fn);
}

const composedFn = composeMany(add1, times3);

console.log(applyToAll(composedFn, 1, 2, 3)); // [add1(times3(1)), add1(times3(2)), add1(times3(3))] => [4, 7, 10]
```

This pattern is powerful for building flexible and reusable data transformations.

## Immediately Invoked Function Expression (IIFE)

An IIFE is a function that runs as soon as it is defined. It helps create a private scope, avoiding polluting the global namespace.

### Syntax

```js
(function () {
  // code here is private
  console.log("IIFE runs immediately!");
})();
```

Or with arrow functions:

```js
(() => {
  console.log("Arrow function IIFE!");
})();
```

### Why Use IIFE?

- Encapsulates variables and functions, preventing conflicts.
- Useful for initialization code.
- Common in older JavaScript patterns before modules.

**Note:** Modern JavaScript modules provide similar encapsulation.
