# Local Storage and Session Storage in the Browser

Modern browsers provide two key mechanisms for storing data on the client side: **Local Storage** and **Session Storage**. Both are part of the Web Storage API and allow you to store key-value pairs in a web browser.

## Local Storage

- **Scope:** Data is stored with no expiration time. It persists even after the browser is closed and reopened.
- **Storage Limit:** Typically around 5-10MB per origin.
- **Use Case:** Storing user preferences, themes, or other data that should persist across sessions.

### Common Functions

```js
// Set item
localStorage.setItem("key", "value");

// Get item
const value = localStorage.getItem("key");

// Remove item
localStorage.removeItem("key");

// Clear all items
localStorage.clear();
```

## Session Storage

- **Scope:** Data is stored for the duration of the page session. It is cleared when the page session ends (tab or window is closed).
- **Storage Limit:** Similar to local storage, but separate for each tab.
- **Use Case:** Storing temporary data like form inputs or state during a single session.

### Common Functions

```js
// Set item
sessionStorage.setItem("key", "value");

// Get item
const value = sessionStorage.getItem("key");

// Remove item
sessionStorage.removeItem("key");

// Clear all items
sessionStorage.clear();
```

## Key Differences

| Feature       | Local Storage          | Session Storage     |
| ------------- | ---------------------- | ------------------- |
| Persistence   | Until manually cleared | Until tab is closed |
| Scope         | Per origin             | Per tab/per window  |
| Storage Limit | ~5-10MB                | ~5-10MB             |
