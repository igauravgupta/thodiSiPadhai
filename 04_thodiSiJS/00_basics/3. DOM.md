## What is the DOM?

- The **DOM (Document Object Model)** is a programming interface for web documents.
- It represents the structure of an HTML or XML document as a tree of objects (nodes).
- Each element, attribute, and piece of text becomes a node in this tree.
- The DOM allows JavaScript to access, modify, add, or remove elements and content dynamically.

## Why is the DOM Important?

- Enables dynamic changes to web pages (content, structure, style)
- Allows user interaction (forms, buttons, events)
- Foundation for modern web development (JS frameworks, interactivity)

## DOM Tree Example

Given this HTML:

```html
<html>
  <body>
    <h1>Hello</h1>
    <p>World</p>
  </body>
</html>
```

The DOM tree looks like:

```
Document
└── html
    └── body
        ├── h1
        └── p
```

## The `window` Object (in short)

- The `window` object is the global object in browsers.
- It represents the browser window/tab and is the top-level object for all browser JS code.
- All global variables and functions become properties of `window`.
- Provides access to:
  - The DOM (`window.document`)
  - Browser features (alerts, prompts, timers, location, history, etc.)
  - Size and screen info (`window.innerWidth`, `window.innerHeight`)

**Common properties/methods:**

- `window.alert()`, `window.prompt()`, `window.confirm()`
- `window.setTimeout()`, `window.setInterval()`
- `window.location` (URL info)
- `window.document` (the DOM)

**Example:**

```js
console.log(window.innerWidth); // width of the viewport
console.log(window.location.href); // current page URL
```

**Note:**

- In browser JS, you can usually omit `window.` (e.g., `alert()` is the same as `window.alert()`).

### The Global Context and `this` Keyword

JavaScript code runs within a context, and in browsers, the global context is the `window` object. All global variables and functions are properties of `window`. The `this` keyword refers to the object that is currently executing the code. In the global context (outside of any function or object), `this` refers to `window`.

**Example:**

```js
var a = 10;
console.log(window.a); // 10
console.log(this.a); // 10 (in global context, 'this' is 'window')
```

When a function is called in the global scope, `this` inside the function also refers to `window`:

```js
function show() {
  console.log(this === window); // true
}
show();
```

However, when a function is called as a method of an object, `this` refers to that object:

```js
const obj = {
  b: 20,
  show: function () {
    console.log(this.b); // 20
  },
};
obj.show();
```

**Summary:**

- The `window` object is the global context in browsers.
- All global variables and functions are properties of `window`.
- The `this` keyword refers to `window` in the global context, but its value depends on how a function is called.

## Selecting Elements: `querySelector` and `querySelectorAll`

- `querySelector` and `querySelectorAll` are modern, flexible methods to select DOM elements using CSS selectors.

### `querySelector`

- Returns the first element that matches a CSS selector.
- If no match, returns `null`.

**Syntax:**

```js
const element = document.querySelector("selector");
```

**Examples:**

```js
const main = document.querySelector("#main"); // by id
const firstItem = document.querySelector(".item"); // by class (first match)
const input = document.querySelector('input[type="text"]'); // by attribute
const body = document.querySelector("body"); // tags
```

### `querySelectorAll`

- Returns a static NodeList of all elements matching the selector.
- If no match, returns an empty NodeList.

**Syntax:**

```js
const elements = document.querySelectorAll("selector");
```

**Example:**

```js
const items = document.querySelectorAll(".item");
items.forEach((el) => (el.style.color = "red"));
```

**When to use?**

- Use `querySelector` for a single (first) match.
- Use `querySelectorAll` for all matches (loop with forEach or for...of).

## Manipulating Any DOM Element

You can change the content, style, and attributes of any HTML element using JavaScript DOM properties and methods.

### 1. Changing Content

- `innerHTML`: Set or get HTML content inside the element (can include HTML tags)
- `innerText` / `textContent`: Set or get plain text content

**Example:**

```js
const el = document.querySelector("div");
el.innerHTML = "<b>Bold content</b>";
el.innerText = "Just text";
el.textContent = "Another text";
```

### 2. Changing Attributes

- Use `setAttribute` and `getAttribute` for any attribute (id, src, href, alt, type, etc.)

```js
el.setAttribute("id", "main");
const id = el.getAttribute("id");
```

### 3. Changing Style

- Use the `style` property to change CSS directly

```js
el.style.color = "blue";
el.style.backgroundColor = "yellow";
```

### 4. Adding/Removing Classes

- Use `classList` to add, remove, or toggle classes

```js
el.classList.add("active");
el.classList.remove("hidden");
el.classList.toggle("highlight");
```

### 5. Working with Data Attributes

- Use `dataset` to get/set custom data attributes (e.g., `data-user-id`)

```js
el.dataset.userId = "123";
console.log(el.dataset.userId); // '123'
```

### 6. Other Common Manipulations

- Remove an element: `el.remove();`
- Add a child: `el.appendChild(childElement);`
- Clone an element: `const copy = el.cloneNode(true);`

---

**Summary:**

- Use `innerHTML`, `innerText`, or `textContent` for content
- Use `setAttribute`/`getAttribute` for any attribute
- Use `style` for inline styles
- Use `classList` for classes
- Use `dataset` for custom data attributes
- Use DOM methods for adding/removing/cloning elements

## JavaScript Dialogs: alert, confirm, prompt

### `alert()`

- Displays a simple modal dialog with a message and an OK button.
- Used to show information or warnings to the user.
- Execution of code pauses until the user dismisses the alert.
  **Syntax:**

```js
alert("This is an alert!");
```

**Example:**

```js
alert("Form submitted successfully!");
```

**Notes:**

- The alert box only shows a message and an OK button (no input).
- It is synchronous (blocks code execution until closed).
- Not suitable for complex UI or frequent notifications (can be annoying).

---

### `confirm()`

- Displays a modal dialog with a message, and OK/Cancel buttons.
- Used to ask the user to confirm an action (e.g., "Are you sure?").
- Returns `true` if OK is pressed, `false` if Cancel is pressed.
  **Syntax:**

```js
const result = confirm("Are you sure you want to delete this?");
```

**Example:**

```js
if (confirm("Do you really want to leave this page?")) {
  alert("You chose OK");
} else {
  alert("You chose Cancel");
}
```

**Notes:**

- Like alert and prompt, confirm is synchronous and blocks code execution until the user responds.
- Use for simple yes/no or confirm/cancel decisions.
- For custom UI, use HTML/CSS modals instead.

---

### `prompt()`

- Displays a modal dialog with a message, an input field, and OK/Cancel buttons.
- Used to get simple input from the user (e.g., name, number).
- Returns the entered value as a string, or `null` if cancelled.
  **Syntax:**

```js
const result = prompt("What is your name?", "Default Value");
```

**Example:**

```js
const age = prompt("Enter your age:");
if (age !== null) {
  alert("You entered: " + age);
}
```

**Notes:**

- Always returns a string (convert to number if needed).
- If the user clicks Cancel, returns `null`.
- Like alert, it is synchronous and blocks code execution.
- Not suitable for secure or complex input (use custom forms for that).

---

**alert vs prompt:**

- `alert()` is for displaying messages only.
- `prompt()` is for getting simple user input.
  **Best Practices:**
- Use sparingly; avoid for critical flows or frequent notifications.
- For better UX, prefer custom modal dialogs using HTML/CSS/JS.

## Selecting Elements by Class Name: `getElementsByClassName`

Use `document.getElementsByClassName` to select all elements with a specific class name.

- Returns a live HTMLCollection of all elements with the given class name.
- If no elements match, returns an empty collection (not null).
- You can access elements by index or loop through the collection.

**Syntax:**

```js
const elements = document.getElementsByClassName("className");
```

**Example:**

```html
<div class="note">Note 1</div>
<div class="note">Note 2</div>
```

```js
const notes = document.getElementsByClassName("note");
for (let i = 0; i < notes.length; i++) {
  notes[i].style.background = "#ff0";
}
```

**When to use?**

- Use `getElementsByClassName` when you want to select all elements with a specific class quickly and don't need advanced CSS selectors.

## Selecting Elements by ID: `getElementById`

The simplest and fastest way to select a single element by its `id` attribute is using `document.getElementById`.

- Returns the element with the specified `id`, or `null` if not found.
- Only one element can have a given `id` in valid HTML.

**Syntax:**

```js
const element = document.getElementById("elementId");
```

**Example:**

```html
<div id="header">Welcome!</div>
```

```js
const header = document.getElementById("header");
header.style.color = "blue";
```

## Multiple Ways to Listen to Events in JavaScript

JavaScript provides several ways to listen for and handle events (like clicks, input, etc.) on DOM elements:

### 1. Inline HTML Event Attribute

Add an event directly in the HTML tag (not recommended for modern code):

```html
<button onclick="alert('Clicked!')">Click Me</button>
```

### 2. Element Property

Assign a function to an element's event property in JavaScript:

```js
const btn = document.getElementById("myBtn");
btn.onclick = function () {
  alert("Button clicked!");
};
```

### 3. `addEventListener` Method (Recommended)

Attach one or more event listeners to an element:

```js
const btn = document.getElementById("myBtn");
btn.addEventListener("click", function () {
  alert("Clicked with addEventListener!");
});
```

// You can add multiple listeners for the same event:
btn.addEventListener("click", () => console.log("Another handler!"));

## Creating Elements Dynamically in JavaScript

You can create new HTML elements dynamically using JavaScript and add them to the DOM.

### 1. Create an Element

Use `document.createElement()` to create a new element:

```js
const newDiv = document.createElement("div");
```

### 2. Set Content or Attributes

You can set text, HTML, classes, ids, or other attributes:

```js
newDiv.innerText = "Hello, I am a new div!";
newDiv.className = "my-class";
newDiv.id = "uniqueId";
```

### 3. Add the Element to the Page

Use `appendChild`, `prepend`, or `insertBefore` to add the new element to the DOM:

```js
document.body.appendChild(newDiv); // Adds at the end of <body>
```

**Add to a specific parent:**

```js
const container = document.getElementById("container");
container.appendChild(newDiv);
```

**Add at the beginning using `prepend`:**

```js
container.prepend(newDiv); // Adds as the first child of container
```

**Insert before another element using `insertBefore`:**

```js
const reference = document.getElementById("referenceElement");
container.insertBefore(newDiv, reference); // Adds before referenceElement
```

### 4. Example: Create and Add a Button

```js
const btn = document.createElement("button");
btn.innerText = "Click Me!";
btn.addEventListener("click", () => alert("Button Clicked!"));
document.body.appendChild(btn);
```
