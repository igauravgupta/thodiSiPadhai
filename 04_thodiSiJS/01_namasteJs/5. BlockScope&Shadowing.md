## Introduction to Blocks in JavaScript

- In JavaScript, a **block** is a section of code enclosed within curly braces `{}`.
- Blocks group statements together, commonly used in control structures like `if`, `for`, and `while`.
- Variables declared with `let` and `const` inside a block are scoped to that block (block scope).

**Example:**

```js
{
  let i = 0; // 'i' is only accessible within this block
}
```

---

## Block Scope and Variable Declarations

### Block Scope

- Variables declared with `let` and `const` are only accessible within the block in which they are defined.
- This prevents them from leaking outside the block.

### `let` and `const` Initialization

- Both `let` and `const` are not hoisted in the same way as `var`. They are in a "temporal dead zone" from the start of the block until the declaration is encountered.
- Accessing them before declaration results in a `ReferenceError`.
- `const` must be initialized at the time of declaration, while `let` can be declared without initialization.

### `var` Behavior

- `var` is function-scoped, not block-scoped. If declared inside a block, it is accessible outside the block if within the same function or global scope.
- `var` declarations are hoisted and initialized with `undefined`.

**Example:**

```js
{
  var a = 1;
  let b = 2;
  const c = 3;
}
console.log(a); // 1
console.log(b); // ReferenceError: b is not defined
console.log(c); // ReferenceError: c is not defined
```

## Shadowing

### Shadowing with `let` and `const`

- **Shadowing** occurs when a variable declared within a certain scope (e.g., a block) has the same name as a variable in an outer scope. The inner variable "shadows" or overrides access to the outer variable within its scope.
- Shadowing with `let` and `const` is limited to the block in which they are declared.
- Shadowing a `var` variable with `let` or `const` in a block does not affect the outer `var` outside the block.
- Redeclaring a variable with `let` or `const` in the same scope is not allowed.

**Example:**

```js
let x = 10;
{
  let x = 20; // Shadows outer 'x'
  console.log(x); // 20
}
console.log(x); // 10
```

### Shadowing `var` with `var`

- Declaring a variable with `var` inside a block that already has a `var` variable with the same name in an outer scope does **not** create a new block-scoped variable.
- The inner declaration refers to the same variable due to `var` being function-scoped.
- The inner `var` declaration will overwrite the value of the outer `var` within the same function or global scope.

**Example:**

```js
var y = 5;
{
  var y = 15; // This does not create a new block-scoped variable
  console.log(y); // 15
}
console.log(y); // 15 (outer 'y' is also changed)
```

### Shadowing `let` with `var` in a Block

- Declaring a `var` variable with the same name as a `let` variable in an outer scope within a block will throw a `SyntaxError`.
- This is because `var` declarations are hoisted to the function or global scope and cannot redeclare a variable that is already declared with `let` in the same scope.

**Example:**

```js
let z = 100;
{
  var z = 200; // SyntaxError: Identifier 'z' has already been declared
}
```

## Legal and Illegal Shadowing

- **Legal Shadowing:**

  - Shadowing variables in JavaScript using `let`, `const`, or `var` in inner scopes is allowed.
  - Function parameters and catch block parameters can shadow variables from outer scopes.

  ```js
  let x = 1;
  {
    let x = 2; // Shadows outer x
  }
  ```

- **Illegal Shadowing:**

  - Redeclaring a variable with `let` or `const` in the same scope is not allowed.

  ```js
  let x = 1;
  let x = 2; // SyntaxError
  ```

  - You cannot shadow a `let` or `const` variable with a `var` in the same or inner scope.

  ```js
  let x = 1;
  {
    var x = 2; // SyntaxError
  }
  ```

  - Function declarations cannot shadow `let` or `const` in the same scope.
