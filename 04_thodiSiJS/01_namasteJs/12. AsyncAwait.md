## Introduction

Async/Await is a modern way to handle asynchronous operations in JavaScript, making code easier to read and maintain.

## What is Async/Await?

- **Async** and **await** are keywords introduced in ES2017.
- They simplify working with Promises by allowing asynchronous code to be written in a synchronous style.

## Why Use Async/Await?

- Improves code readability.
- Reduces callback hell and chaining.
- Easier error handling with `try...catch`.

## How Async/Await Works

- An `async` function always returns a Promise.
- The `await` keyword pauses the execution of the function until the Promise resolves or rejects.

## Async Functions

```js
async function fetchData() {
  // function body
}
```

- Declared with the `async` keyword.
- Always returns a Promise.

## Await Keyword

```js
const data = await fetch("https://api.example.com/data");
```

- Can only be used inside `async` functions.
- Waits for the Promise to resolve and returns the result.

## Error Handling

Use `try...catch` blocks for error handling:

```js
async function getData() {
  try {
    const response = await fetch("https://api.example.com/data");
    const data = await response.json();
    return data;
  } catch (error) {
    console.error("Error:", error);
  }
}
```

## Async/Await vs Promises

| Feature        | Promises                | Async/Await               |
| -------------- | ----------------------- | ------------------------- |
| Syntax         | Chained `.then()` calls | Synchronous-like          |
| Error Handling | `.catch()`              | `try...catch`             |
| Readability    | Can get nested          | Cleaner, easier to follow |

## Common Pitfalls

- `await` only works inside `async` functions.
- Multiple awaits run sequentially unless handled with `Promise.all`.
- Forgetting to handle errors.

## Best Practices

- Use `try...catch` for error handling.
- Use `Promise.all` for parallel async operations.
- Avoid blocking the main thread with long-running awaits.

## Examples

**Sequential Await:**

```js
async function sequential() {
  const user = await getUser();
  const posts = await getPosts(user.id);
  return posts;
}
```

**Parallel Await:**

```js
async function parallel() {
  const [user, posts] = await Promise.all([getUser(), getPosts()]);
  return { user, posts };
}
```

## Conclusion

Async/Await makes asynchronous JavaScript code easier to write, read, and maintain. Use it to simplify your async workflows and improve code quality.
