# Temporal Dead Zone (TDZ), `let`, and `const` Hoisting

## What is the Temporal Dead Zone?

The **Temporal Dead Zone (TDZ)** is the period between entering a scope (like a block or function) and the point where a variable declared with `let` or `const` is initialized. Accessing the variable in this zone results in a `ReferenceError`.

```js
console.log(a); // ReferenceError: Cannot access 'a' before initialization
let a = 10;
```

## `let` and `const` Hoisting

- **Hoisting** means variable declarations are moved to the top of their scope during the compilation phase.
- Variables declared with `var` are hoisted and initialized with `undefined`.
- Variables declared with `let` and `const` are hoisted but **not initialized**. They remain uninitialized until their declaration is evaluated, creating the TDZ.

```js
console.log(b); // ReferenceError
let b = 20;

console.log(c); // ReferenceError
const c = 30;
```

## Key Points

- `let` and `const` are hoisted but not initialized.
- Accessing them before declaration causes a `ReferenceError` (TDZ).
- Always declare variables at the top of their scope to avoid TDZ issues.

## Errors Related to Variable Declarations

### SyntaxError

A `SyntaxError` occurs when the code violates JavaScript's syntax rules, such as redeclaring a variable with `let` or `const` in the same scope.

```js
let d = 1;
let d = 2; // SyntaxError: Identifier 'd' has already been declared
```

### ReferenceError

A `ReferenceError` is thrown when trying to access a variable that is in the TDZ or not declared at all.

```js
console.log(e); // ReferenceError: Cannot access 'e' before initialization
let e = 5;
```

### TypeError

A `TypeError` occurs when an operation is performed on a value of the wrong type, such as attempting to reassign a `const` variable.

```js
const f = 10;
f = 20; // TypeError: Assignment to constant variable.
```
