## Different Types of Function Declarations in JavaScript

JavaScript provides several ways to declare functions. Here are the most common types, with syntax and examples:

### 1. Function Declaration (Function Statement)

```js
function greet(name) {
  return "Hello, " + name + "!";
}
greet("Alice");
```

- **Hoisted**: Can be called before its definition.

### 2. Function Expression

```js
const greet = function (name) {
  return "Hello, " + name + "!";
};
greet("Bob");
```

- **Not hoisted**: Cannot be called before its definition.

### 3. Arrow Function Expression

```js
const greet = (name) => {
  return "Hello, " + name + "!";
};
// Or, for single expressions:
const add = (a, b) => a + b;
```

- **Shorter syntax**
- **No own `this`**

### 4. Anonymous Function

- A function without a name, often used as a callback:

```js
setTimeout(function () {
  console.log("Hello from anonymous function");
}, 1000);
```

### 5. Named Function Expression

```js
const c = function foo() {
  console.log("Named function expression");
  // foo(); // valid: can call itself recursively
};
c();
// foo(); // ReferenceError: foo is not defined outside the function
```

### 6. Immediately Invoked Function Expression (IIFE)

```js
(function () {
  console.log("IIFE runs immediately!");
})();
```

- Runs as soon as it is defined.

### 7. Function Constructor (Not Recommended)

```js
const sum = new Function("a", "b", "return a + b");
console.log(sum(2, 3)); // 5
```

- Rarely used; avoid for security and performance reasons.

**Tip:** Use function declarations or arrow functions for most use cases. Prefer `const` for function expressions to avoid accidental reassignment.

## 1. Function Statement (Function Declaration)

A function statement (or declaration) defines a named function using the `function` keyword. It is hoisted, so you can call it before its definition in the code.

```js
function a() {
  console.log("a called");
}
a(); // works even if called before the function definition
```

---

## 2. Function Expression

A function expression is when a function is assigned to a variable. It can be anonymous or named. Function expressions are not hoisted, so you cannot call them before their definition.

```js
var b = function () {
  console.log("b called");
};
b();
```

---

## 3. Anonymous Function

An anonymous function is a function without a name. It is often used as a value (e.g., in function expressions or as arguments to other functions).

```js
setTimeout(function () {
  console.log("Hello from anonymous function");
}, 1000);
```

// Invalid as a standalone statement:
// function () {} // SyntaxError

---

## 4. Named Function Expression

A named function expression is a function expression with a name. The name is only accessible inside the function itself (useful for recursion or debugging).

```js
var c = function foo() {
  console.log("Named function expression");
  // foo(); // valid: can call itself recursively
};
c();
// foo(); // ReferenceError: foo is not defined outside the function
```

## Difference between Parameters & Arguments

- **Parameters** are the variables listed in the function definition.
- **Arguments** are the actual values passed to the function when it is called.

```js
function sum(a, b) {
  // a, b are parameters
  return a + b;
}
sum(2, 3); // 2, 3 are arguments
```

## Functions That Take Unlimited Arguments

JavaScript functions can accept any number of arguments, even if not specified in the parameter list. There are two main ways to handle unlimited arguments:

### 1. The `arguments` object

- The `arguments` object is an array-like object available inside all non-arrow functions. It contains all arguments passed to the function.

```js
function sumAll() {
  let total = 0;
  for (let i = 0; i < arguments.length; i++) {
    total += arguments[i];
  }
  return total;
}
console.log(sumAll(1, 2, 3, 4)); // 10
```

### 2. Using Rest Parameters (`...args`)

- Modern and preferred way. Collects all remaining arguments into an array.

```js
function sumAll(...args) {
  return args.reduce((acc, curr) => acc + curr, 0);
}
console.log(sumAll(1, 2, 3, 4)); // 10
```

**Interview Tip:** Use rest parameters for new code. Use `arguments` if you need to support old code or want to see how legacy code works.

## First Class Functions

In JavaScript, functions are first-class citizens. This means:

- Functions can be assigned to variables
- Passed as arguments to other functions
- Returned from other functions

```js
function greet() {
  return "Hello";
}

function callFunction(fn) {
  console.log(fn());
}

callFunction(greet); // Passing function as argument

function outer() {
  return function () {
    console.log("Returned from another function");
  };
}
outer()(); // Returning a function from a function
```

## Arrow Functions & `this`

Arrow functions are a concise way to write functions in ES6+. They do not have their own `this`, `arguments`, or `super`.

```js
const add = (a, b) => a + b;
console.log(add(2, 3)); // 5

const greet = () => {
  console.log("Hello from arrow function");
};
greet();
```

### The `this` Keyword in Arrow Functions

- **Arrow functions do not have their own `this` context.**
- Instead, `this` inside an arrow function refers to the value of `this` in the scope where the arrow function was created (lexical scoping).
- In contrast, regular functions get their own `this` depending on how they are called (object method, event handler, etc).

#### Example: `this` in Arrow vs Regular Function

```js
const obj = {
  value: 42,
  regularFunc: function () {
    console.log(this.value); // 42 (this refers to obj)
  },
  arrowFunc: () => {
    console.log(this.value); // undefined (this refers to global or enclosing scope)
  },
};

obj.regularFunc(); // 42
obj.arrowFunc(); // undefined
```

**Summary:**

- Use regular functions when you need your own `this` (e.g., object methods, event handlers).
- Use arrow functions for callbacks or when you want to inherit `this` from the surrounding scope.

## Higher Order Functions and Callbacks (Detailed)

### What is a Higher Order Function?

A **higher order function** is a function that does at least one of the following:

- Takes one or more functions as arguments (parameters)
- Returns a function as its result

#### Key Characteristics:

- Enables functional programming style
- Promotes code reusability and abstraction
- Common in array methods, event handling, and async code

#### Real-World Examples:

1. **Array Methods**
   - `map`, `filter`, `reduce`, `forEach` are all higher order functions because they take a callback function as an argument.
   ```js
   const nums = [1, 2, 3, 4];
   const doubled = nums.map((x) => x * 2); // [2, 4, 6, 8]
   const even = nums.filter((x) => x % 2 === 0); // [2, 4]
   const sum = nums.reduce((acc, curr) => acc + curr, 0); // 10
   ```
2. **Event Listeners**
   ```js
   document.addEventListener("click", function () {
     console.log("Clicked!");
   });
   ```
3. **Function Returning Function**
   ```js
   function greetMaker(greeting) {
     return function (name) {
       return greeting + ", " + name;
     };
   }
   const sayHello = greetMaker("Hello");
   console.log(sayHello("Alice")); // Hello, Alice
   ```

### What is a Callback Function?

A **callback** is a function passed as an argument to another function, to be executed later (after some operation or event).

#### Types of Callbacks:

- **Synchronous**: Executed immediately (e.g., `forEach`, `map`)
- **Asynchronous**: Executed after an operation completes (e.g., `setTimeout`, event handlers, AJAX)

#### Examples:

```js
// Synchronous callback
[1, 2, 3].forEach(function (x) {
  console.log(x);
});

// Asynchronous callback
setTimeout(function () {
  console.log("Runs after 1 second");
}, 1000);
```

#### Why are Callbacks Important?

- Enable custom logic to be injected into generic functions
- Essential for handling asynchronous operations (timers, events, network requests)
- Foundation for Promises and async/await
