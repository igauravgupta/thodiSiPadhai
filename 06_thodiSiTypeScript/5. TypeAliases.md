# Type Aliases — TypeScript (and use cases in some languages)

## What is a type alias (TypeScript)

A type alias names any type—primitive, union, tuple, function, mapped, conditional, etc.—so you can reuse and compose complex types.

Syntax:

```ts
type Name = ...;
```

## Common patterns in TypeScript

- Primitive alias

```ts
type ID = string;
```

- Union / discriminated unions

```ts
type Status = "idle" | "loading" | "error";
type Action = { type: "add"; payload: string } | { type: "remove"; id: ID };
```

- Object / tuple / function

```ts
type Point = { x: number; y: number };
type Vec = [number, number];
type Handler = (e: Event) => void;
```

- Intersection types
  Combine multiple types into one using `&`. Useful for composing object shapes, adding behaviors, or creating overloaded function signatures.

  - Basic object intersection

  ```ts
  type A = { a: number };
  type B = { b: string };
  type AB = A & B; // { a: number; b: string }
  const x: AB = { a: 1, b: "two" };
  ```

  - Overlapping properties

  ```ts
  type X = { id: string };
  type Y = { id: string | number };
  type XY = X & Y; // { id: string }  (intersection narrows to common assignable type)
  ```

  - Intersections with unions (distribution)

  ```ts
  type U = { kind: "a"; a: number } | { kind: "b"; b: string };
  type Named = { name: string };
  type NamedU = U & Named; // ({kind:"a";a:number;name:string} | {kind:"b";b:string;name:string})
  ```

  - Function intersections (create overloaded signatures)

  ```ts
  type F = ((x: number) => string) & ((x: string) => number);

  // A value of type F can be implemented as an overloaded function
  const f: F = ((x: any) =>
    typeof x === "number" ? String(x) : Number(x)) as F;
  ```

  - Use cases
    - Compose reusable traits (mixins) without inheritance.
    - Add metadata or capabilities to existing types.
    - Express overloaded call signatures in types.
    - Narrow combined property requirements when composing APIs.

## Example snippets (runnable ideas)

- Using a discriminated union with exhaustive switch

```ts
type Result =
  | { status: "ok"; value: number }
  | { status: "err"; message: string };

function handle(r: Result) {
  switch (r.status) {
    case "ok":
      return r.value * 2;
    case "err":
      console.error(r.message);
      return null;
    default:
      // ensure exhaustiveness
      const _exhaustive: never = r;
      return _exhaustive;
  }
}
```
