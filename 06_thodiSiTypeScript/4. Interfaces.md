# Interfaces

Define object shapes and contracts in TypeScript.

## Purpose

- Describe the shape of objects, functions, and classes.
- Provide a compile-time contract (structural typing).
- Enable tooling (autocomplete, refactoring, documentation).

## Basic syntax

```ts
interface User {
  id: number;
  name: string;
}

const u: User = { id: 1, name: "Alice" };
```

## Common features

- Optional properties

```ts
interface Config {
  timeout?: number;
}
```

- Readonly properties

```ts
interface Point {
  readonly x: number;
  readonly y: number;
}
```

- Index signatures

```ts
interface StringMap {
  [key: string]: string;
}
```

- Function / call signatures

```ts
interface Adder {
  (a: number, b: number): number;
}
```

- Construct signatures

```ts
interface Ctor {
  new (name: string): Object;
}
```

- Extending interfaces

```ts
interface Animal {
  name: string;
}
interface Dog extends Animal {
  bark(): void;
}
```

- Implementing in classes

```ts
class Person implements User {
  constructor(public id: number, public name: string) {}
}
```

- Declaration merging (interfaces with same name merge)

```ts
interface A {
  x: number;
}
interface A {
  y: number;
} // result: A has x and y
```
