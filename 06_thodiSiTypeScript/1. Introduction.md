# About TypeScript

TypeScript is a typed superset of JavaScript that compiles to plain JavaScript. It adds optional static types, interfaces, and tooling that help catch errors early, improve refactoring, and provide better editor/autocomplete support.

## Install TypeScript

Install locally as a dev dependency (recommended):

```bash
npm install --save-dev typescript
```

Or install the compiler globally:

```bash
npm install --global typescript
```

Useful utilities:

```bash
npm install --save-dev ts-node @types/node   # run TS directly and Node type definitions
npm install --save-dev typescript eslint      # linting for TS projects
```

## Recommended Editor Extensions

1. ESLint (with TypeScript support)
2. TypeScript and JavaScript Nightly
3. Prettier (optional) — for consistent formatting

## Project setup

1. Initialize project:

```bash
npm init -y
npm install --save-dev typescript
npx tsc --init
```

2. Purpose of `tsconfig.json`: configure the TypeScript compiler (target, module system, strictness, output directory, includes/excludes, etc.).

3. Minimal `tsconfig.json` example:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "outDir": "dist",
    "rootDir": "src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src"]
}
```

4. Typical `package.json` scripts:

```json
{
  "scripts": {
    "build": "tsc",
    "watch": "tsc --watch",
    "start": "node dist/index.js",
    "dev": "ts-node src/index.ts"
  }
}
```

5. Suggested folder layout:

```
project/
├─ src/
│  └─ index.ts
├─ dist/
├─ package.json
└─ tsconfig.json
```

6. Basic workflow:

- Write source in `src/*.ts`
- Compile: `npm run build`
- Dev: `npm run watch` or `npm run dev` (with ts-node)

That should get a basic TypeScript project up and running.

## Use-case Q&A: common `rootDir` / `outDir` errors and fixes

Q: I uncommented `rootDir` and `outDir` in `tsconfig.json` and `npx tsc` now shows "Source file is not under 'rootDir'". Why?

A: `rootDir` tells the compiler where your input TypeScript files live. If any `.ts` file that the compiler is asked to compile is outside that directory, TypeScript will complain. Common causes:

- Your `include` glob (or `files`) references files not under `rootDir`.
- You have source files at the project root but set `rootDir` to `src`.

Fix: either move sources under the declared `rootDir`, update `rootDir` to the correct folder (or remove it), and ensure `include` matches. Example quick checks:

```bash
# run the project build and see the exact error
npx tsc -p tsconfig.json

# show where your .ts files are
find . -type f -name "*.ts" -maxdepth 6

# print tsconfig to inspect rootDir/outDir/include
cat tsconfig.json
```

Q: I see an error that output files would overwrite input, or compiled files appear inside my `src` folder. What's wrong?

A: That usually means `outDir` is set to a path that overlaps `rootDir` (for example `outDir: "src"` or `outDir` inside `rootDir`). Set `outDir` to a separate directory (commonly `dist` or `build`) and exclude it from the compile `include`.

Example safe setting (source in `src/`, compiled in `dist/`):

```json
{
  "compilerOptions": {
    "rootDir": "src",
    "outDir": "dist",
    "target": "ES2020",
    "module": "commonjs"
  },
  "include": ["src/**/*"],
  "exclude": ["dist"]
}
```

Q: I have `.ts` files in multiple places (project root and `src/`). How do I configure `rootDir`?

A: If sources are scattered you can either:

- Move sources into a single directory like `src/` (recommended), or
- Omit `rootDir` and let the compiler infer the common directory, or
- Explicitly set `rootDir` to `.` (project root) and ensure `outDir` is separate.

Example when keeping files at project root and `src/`:

```json
{
  "compilerOptions": {
    "rootDir": ".",
    "outDir": "dist"
  },
  "include": ["src", "*.ts"],
  "exclude": ["dist", "node_modules"]
}
```

Q: Any quick troubleshooting checklist?

A: Yes—try these in order:

1. Run `npx tsc -p tsconfig.json` and copy the exact compiler message.
2. Run `find . -name "*.ts"` to list where your sources are.
3. Confirm `include` globs in `tsconfig.json` actually match those files.
4. Make sure `outDir` is not the same as or inside `rootDir`.
5. If unsure, remove `rootDir` and `outDir` temporarily to let tsc infer paths, then re-add them once you confirm the layout.

Q: Example real-world scenario

A: Given this layout:

```
project/
├─ src/
│  └─ index.ts
├─ scripts/
│  └─ migrate.ts
├─ package.json
└─ tsconfig.json
```

If you want everything compiled into `dist/`, use:

```json
{
  "compilerOptions": {
    "rootDir": ".",
    "outDir": "dist",
    "module": "commonjs",
    "target": "ES2019"
  },
  "include": ["src/**/*", "scripts/**/*"]
}
```

Q: Anything else I should watch for?

A: A few more tips:

- Always exclude `outDir` (and `node_modules`) to avoid re-compiling generated files.
- Use local TypeScript (devDependency) and run `npx tsc` or npm scripts so everyone uses the same compiler version.
- In VS Code, choose the workspace TypeScript version (Command Palette → "Select TypeScript Version").

Try this small command after fixing `tsconfig.json` to verify the build:

```bash
npx tsc -p tsconfig.json
```
