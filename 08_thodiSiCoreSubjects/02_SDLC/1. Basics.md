# Software Development Life Cycle (SDLC)

## What is SDLC?

SDLC stands for Software Development Life Cycle. It is a systematic process for planning, creating, testing, deploying, and maintaining software applications.

## Why Do We Need SDLC?

- Ensures a structured approach to software development
- Improves project management and control
- Reduces risks and errors
- Enhances product quality
- Facilitates communication among stakeholders

## Phases in SDLC

1. **Requirement Analysis**
2. **System Design**
3. **Implementation (Coding)**
4. **Testing**
5. **Deployment**
6. **Maintenance**

## Importance of SDLC

- Provides a clear project roadmap
- Helps deliver software on time and within budget
- Ensures software meets user requirements
- Supports continuous improvement and maintenance

## Names of All SDLC Models

- Waterfall Model
- V-Model
- Incremental Model
- Iterative Model
- Spiral Model
- Big Bang Model
- Agile Model
- RAD (Rapid Application Development) Model
- Prototype Model

## Factors for Choosing an SDLC Model

- Project size and complexity
- Budget and timeline constraints
- Client requirements and involvement
- Team expertise and experience
- Flexibility for changes
- Risk assessment and management
- Technology stack and tools
