# SDLC Phases and Their Details

## 1. Requirement Analysis

- Gather and document user requirements.
- Identify project goals, constraints, and expectations.
- Stakeholder meetings and feasibility studies.
- Distinguish between functional and non-functional requirements:
  - **Functional requirements** specify what the system should do (features, behaviors, tasks).
  - **Non-functional requirements** define system qualities (performance, security, usability, reliability).
- Document both types to ensure complete understanding of user needs.

## 2. System Design

- Create architecture and design specifications.
- Define hardware, software, and system requirements.
- Prepare data models, interface designs, and workflows.
- Distinguish between High-Level Design (HLD) and Low-Level Design (LLD):
  - **High-Level Design (HLD):**
    - Focuses on system architecture, modules, and their interactions.
    - Provides an overview of the solution, including technology choices and major components.
    - Used by architects and senior developers for planning.
  - **Low-Level Design (LLD):**
    - Details the internal logic of individual modules and components.
    - Specifies algorithms, data structures, and interface details.
    - Used by developers for actual coding and implementation.
- Both HLD and LLD are essential for translating requirements into a working system.

## 3. Implementation (Coding)

- Developers write code based on design documents.
- Use chosen programming languages and tools.
- Perform unit testing during development.

## 4. Testing

- Validate the system against requirements.
- Conduct various tests: unit, integration, system, acceptance.
- Identify and fix defects.

### Types of Testing

- **Unit Testing:**  
   Tests individual components or functions to ensure they work as intended. Usually performed by developers during coding.

- **Integration Testing:**  
   Checks the interaction between integrated modules or components to verify they work together correctly.

- **System Testing:**  
   Validates the complete and integrated system to ensure it meets specified requirements.

- **Acceptance Testing:**  
   Confirms the system meets business needs and requirements. Often performed by end users or stakeholders.

- **Regression Testing:**  
   Ensures that new changes or bug fixes do not negatively affect existing functionality.

- **Smoke Testing:**  
   Performs basic tests to check if the major functionalities of the system work, often used after a new build.

- **Sanity Testing:**  
   Verifies specific functionality or bug fixes after changes, focusing on a narrow area.

- **Performance Testing:**  
   Assesses system speed, responsiveness, and stability under load.

- **Security Testing:**  
   Identifies vulnerabilities and ensures data protection and system security.

- **Usability Testing:**  
   Evaluates the system’s user interface and overall user experience.

- **Compatibility Testing:**  
   Checks if the system works across different devices, operating systems, browsers, or environments.

Each type of testing helps ensure the software is robust, reliable, and ready for deployment.

## 5. Deployment

- Release the system to the production environment.
- Perform installation, configuration, and user training.
- Monitor initial usage and resolve issues.

## 6. Maintenance

- Provide ongoing support and updates.
- Fix bugs, enhance features, and ensure system reliability.
- Adapt system to changing requirements or environments.
