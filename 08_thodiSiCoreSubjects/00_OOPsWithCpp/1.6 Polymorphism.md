# Polymorphism in C++

Polymorphism is a core concept in Object-Oriented Programming (OOP) that allows objects to be treated as instances of their parent class rather than their actual class. It enables one interface to be used for a general class of actions.

Polymorphism is an Object-Oriented Programming (OOP) feature that allows the same function, operator, or object to behave differently in different contexts.
The word comes from “poly” (many) + “morph” (forms) → “many forms.”

## Why Do We Need Polymorphism?

- **Code Reusability:** Write generic code that works with different types.
- **Flexibility:** Change behavior at runtime.
- **Maintainability:** Easier to extend and modify code.

## Types of Polymorphism

1. **Compile-Time Polymorphism (Static)**
   - Implemented during compilation.
   - Achieved through function overloading and operator overloading.
2. **Run-Time Polymorphism (Dynamic)**
   - Implemented during program execution.
   - Achieved using inheritance and virtual functions.

---

## How to Achieve Polymorphism

### 1. Compile-Time Polymorphism

#### Function Overloading or Method Overloading

**Definition:**  
Function overloading in C++ allows multiple functions to share the same name, provided they differ in the type or number of their parameters. The compiler determines which function to call based on the arguments supplied.

**Rules for Function Overloading:**

- Functions must have different parameter types or counts.
- Overloading cannot be done by return type alone.
- Using default arguments may lead to ambiguity if not managed properly.
- All overloaded functions must be declared in the same scope.

```cpp
class Print {
public:
     void show(int i) { std::cout << "Integer: " << i << std::endl; }
     void show(double d) { std::cout << "Double: " << d << std::endl; }
};
```

#### Operator Overloading

```cpp
class Complex {
public:
     int real, imag;
     Complex operator + (const Complex& obj) {
          Complex temp;
          temp.real = real + obj.real;
          temp.imag = imag + obj.imag;
          return temp;
     }
};
```

### 2. Run-Time Polymorphism

#### Using Function Overriding

**Definition:**  
Function overriding occurs when a derived class provides its own implementation of a function that is already defined in its base class. The overridden function in the derived class must have the same signature as in the base class.

**Example:**

```cpp
class Animal {
public:
    void speak() {
        std::cout << "Animal speaks" << std::endl;
    }
};

class Dog : public Animal {
public:
    void speak() {
        std::cout << "Dog barks" << std::endl;
    }
};
```

Here, `Dog` overrides the `speak()` function of `Animal`. When called on a `Dog` object, it executes the derived version.

#### Using Virtual Functions

```cpp
class Base {
public:
     virtual void display() { std::cout << "Base display" << std::endl; }
};

class Derived : public Base {
public:
     void display() override { std::cout << "Derived display" << std::endl; }
};

void show(Base* b) {
     b->display();
}
```

When `show()` is called with a `Derived` object, the `Derived` version of `display()` is executed.
The call to the function is determined at runtime -> also known as runtime Polymorphism.

---

## Summary

- **Polymorphism** allows objects to be accessed in multiple forms.
- **Types:** Compile-time (function/operator overloading), Run-time (virtual functions).
- **Benefits:** Flexibility, reusability, maintainability.
- **Implementation:** Use overloading for compile-time, virtual functions for run-time.
