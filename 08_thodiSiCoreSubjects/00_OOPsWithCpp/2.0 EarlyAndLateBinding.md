## Introduction

👉 Binding means deciding which function definition will be called when a function is invoked.
It answers: “Which function code should run for this function call?”

---

## Early Binding (Static Binding / Compile-time Binding)

The function call is resolved at compile time.
The compiler knows exactly which function to call.
Faster (no runtime overhead).

Achieved by:

Normal function calls
Function overloading
Operator overloading
Non-virtual functions

### Early Binding Example

✅ Example:

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    void display() { cout << "Base Display\n"; }  // non-virtual
};

class Derived : public Base {
public:
    void display() { cout << "Derived Display\n"; }
};

int main() {
    Base b;
    Derived d;

    b.display();  // Base Display (decided at compile time)
    d.display();  // Derived Display (decided at compile time)

    Base* ptr = &d;
    ptr->display();  // Base Display (still early binding, because not virtual)
}
```

---

## Late Binding (Dynamic Binding / Runtime Binding)

The function call is resolved at runtime.
The compiler does not know in advance which function will be called.
Achieved using virtual functions (with base class pointers or references).
Implemented using vtable (virtual table) mechanism.

### Late Binding Example

✅ Example:

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void display() { cout << "Base Display\n"; }  // virtual function
};

class Derived : public Base {
public:
    void display() override { cout << "Derived Display\n"; }
};

int main() {
    Base* ptr;
    Derived d;

    ptr = &d;
    ptr->display();  // Derived Display (decided at runtime, late binding)
}
```

## Key Difference

🔁 Key Difference

| Feature       | Early Binding (Static)                               | Late Binding (Dynamic)       |
| ------------- | ---------------------------------------------------- | ---------------------------- |
| Decision Time | Compile-time                                         | Runtime                      |
| Speed         | Faster (no overhead)                                 | Slower (vtable lookup)       |
| Achieved By   | Function/Operator overloading, Non-virtual functions | Virtual functions            |
| Flexibility   | Less flexible                                        | More flexible (polymorphism) |

---

## One-liner exam definition

✅ One-liner exam definition:

Early Binding: Function call is resolved at compile time.

Late Binding: Function call is resolved at runtime using virtual functions.
