# Access Specifiers in C++

Access specifiers in C++ determine the accessibility of class members (variables and functions). They control how and where the members can be accessed.

## Types of Access Specifiers

### 1. `public`

- Members declared as `public` are accessible from outside the class.
- They can be accessed by any function.

```cpp
class Example {
public:
    int x; // public member
};
```

### 2. `private`

- Members declared as `private` are accessible only within the class.
- They cannot be accessed directly from outside the class.

```cpp
class Example {
private:
    int y; // private member
};
```

### 3. `protected`

- Members declared as `protected` are accessible within the class and by derived classes.
- They are not accessible from outside the class.

```cpp
class Example {
protected:
    int z; // protected member
};
```

## Example

```cpp
class Base {
public:
    int a;
private:
    int b;
protected:
    int c;
};

class Derived : public Base {
public:
    void show() {
        a = 10;    // OK: public
        // b = 20; // Error: private
        c = 30;    // OK: protected
    }
};
```

## Summary Table

| Specifier | Accessible within class | Accessible by derived class | Accessible outside class |
| --------- | :---------------------: | :-------------------------: | :----------------------: |
| public    |           Yes           |             Yes             |           Yes            |
| private   |           Yes           |             No              |            No            |
| protected |           Yes           |             Yes             |            No            |

## Notes

- By default, members of a class are `private`.
- Access specifiers help implement encapsulation and data hiding in C++.
