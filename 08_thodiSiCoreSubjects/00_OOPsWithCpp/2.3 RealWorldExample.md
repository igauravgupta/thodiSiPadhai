## Class & Object

Class: Blueprint/template
Example: Car is a class.

Object: Real instance of class
Example: Car myCar = new Car("Tesla", "Model S");

```cpp
class Car {
public:
    string brand;
    string model;
    int speed;

    Car(string b, string m) {
        brand = b;
        model = m;
        speed = 0;
    }

    void accelerate() {
        speed += 10;
    }
};
```

Here Car is a class.
myCar or car1 are objects.

## Encapsulation (Data Hiding)

Wrapping data + methods together.
Example: We don’t want anyone to directly change speed to 5000. Instead, we control it via methods (accelerate, brake).

```cpp
class Car {
private:
    int speed; // hidden data
public:
    void setSpeed(int s) {
        if(s >= 0 && s <= 300) speed = s; // validation
    }
    int getSpeed() {
        return speed;
    }
};
```

## Inheritance (Code Reusability)

Child classes inherit features from parent.
Example: ElectricCar inherits from Car.

```cpp
class ElectricCar : public Car {
public:
    int batteryLevel;
    void charge() {
        batteryLevel = 100;
    }
};
```

ElectricCar automatically has speed, setSpeed(), etc., from Car.

## Polymorphism (Many Forms)

### Compile-time (Overloading):

```cpp
class Car {
public:
    void drive() { cout << "Driving normally\n"; }
    void drive(int speed) { cout << "Driving at " << speed << " km/h\n"; }
};
```

### Runtime (Overriding with Virtual Functions):

```cpp
class Car {
public:
    virtual void fuelType() {
        cout << "Petrol or Diesel\n";
    }
};

class ElectricCar : public Car {
public:
    void fuelType() override {
        cout << "Electric Battery\n";
    }
};

Car* c1 = new Car();
Car* c2 = new ElectricCar();

c1->fuelType(); // Petrol or Diesel
c2->fuelType(); // Electric Battery
```

## Abstraction (Hiding Implementation, Showing Functionality)

User only knows what the function does, not how.
Example: Abstract Class / Interface

```cpp
class Vehicle {
public:
    virtual void start() = 0; // pure virtual function
};

class Car : public Vehicle {
public:
    void start() override {
        cout << "Car starts with a key or button\n";
    }
};
```

The driver just knows start() — doesn’t care about the engine code inside.

## How to Explain in an Interview

👉 Use the "Car System" example like this:

- Class/Object → Car is class, myCar is object.
- Encapsulation → Keep speed private, control access.
- Inheritance → ElectricCar inherits Car.
- Polymorphism → Same fuelType() behaves differently for petrol/diesel/electric.
- Abstraction → Driver just calls start(), no need to know engine details.
