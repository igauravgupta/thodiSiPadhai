# Abstraction in OOP (C++)

Abstraction is a core principle of object-oriented programming (OOP) that enables developers to manage complexity by exposing only the essential features of an object, while hiding the underlying implementation details. This allows programmers to focus on what an object does, rather than how it does it.

Focuses on what an object does rather than how it does it.

## Why Do We Need Abstraction?

- **Simplifies Complex Systems:**  
   Abstraction helps break down complex systems into manageable components by modeling classes based on relevant attributes and behaviors.
- **Manages Complexity:**  
   Developers can interact with high-level interfaces, avoiding the need to understand or modify low-level implementation details.
- **Enhances Maintainability and Scalability:**  
   By reducing dependencies on specific implementations, abstraction makes code easier to maintain, extend, and refactor.

## How Is Abstraction Achieved in C++?

- **Abstract Classes and Interfaces:**  
   Define abstract classes (classes with at least one pure virtual function) and interfaces to specify what operations can be performed, without dictating how they are implemented.
  ```cpp
  class Shape {
  public:
          virtual void draw() = 0; // Pure virtual function
  };
  ```
- **Access Modifiers:**  
   Use `private`, `protected`, and `public` access specifiers to restrict access to class members, exposing only what is necessary.
- **Designing Focused Methods and Properties:**  
   Provide methods and properties that represent only the required functionality, hiding unnecessary details from the user.

## Abstraction vs. Encapsulation

| Aspect  | Abstraction                                         | Encapsulation                                     |
| ------- | --------------------------------------------------- | ------------------------------------------------- |
| Purpose | Hides complexity by exposing only relevant features | Bundles data and methods, restricts direct access |
| Focus   | "What" an object does                               | "How" an object achieves its functionality        |
| Example | Abstract classes, interfaces                        | Private data members, getter/setter methods       |

- **Abstraction:**  
   Focuses on exposing only the necessary operations and attributes, hiding the implementation details.
- **Encapsulation:**  
   Protects the internal state of an object by restricting direct access and providing controlled interfaces.

## Example

```cpp
class Vehicle {
public:
        virtual void startEngine() = 0; // Abstract method
};

class Car : public Vehicle {
public:
        void startEngine() override {
                // Implementation details hidden from user
                std::cout << "Car engine started." << std::endl;
        }
};
```

In this example, `Vehicle` provides an abstract interface, while `Car` implements the details. The user interacts with `startEngine()` without needing to know how the engine is started.

---

**Summary:**  
Abstraction is essential for designing robust, maintainable, and scalable software systems. It allows developers to work with simplified models and high-level interfaces, making code easier to understand and modify.

Encapsulation = How you bundle + protect data (data hiding).

Abstraction = How you show only necessary behavior (implementation hiding).
