## Difference Between Function-Oriented and Object-Oriented Programming

**Function-Oriented Programming** focuses on dividing the program into functions. Each function performs a specific task, and data is passed between functions. The main emphasis is on the functions and the flow of data.

**Object-Oriented Programming (OOP)** organizes code around objects, which combine data and behavior. Objects interact with each other, and the main focus is on encapsulating data and methods together.

| Aspect        | Function-Oriented Programming | Object-Oriented Programming       |
| ------------- | ----------------------------- | --------------------------------- |
| Structure     | Functions                     | Objects and Classes               |
| Data Handling | Shared, global                | Encapsulated in objects           |
| Reusability   | Through functions             | Through inheritance, polymorphism |
| Modularity    | Limited                       | High                              |

## Why We Need Modular Programming

Modular programming is essential because it breaks complex programs into smaller, manageable modules. Each module handles a specific responsibility, making code easier to read, maintain, test, and reuse. It also improves collaboration among developers and helps in debugging by isolating issues within modules.

## Classes and Objects in OOP

In Object-Oriented Programming, a **class** is a blueprint for creating objects. It defines the properties (variables) and behaviors (methods) that the objects created from the class will have. An **object** is an instance of a class, representing a specific run-time or real world entity with its own values for the properties defined by the class.

Classes help in encapsulating data and functions together, promoting modularity and reusability. Objects interact with each other using methods, allowing for organized and maintainable code.

Objects have a state, represented by their variables, and behavior, defined by their methods.

### Example: C++ Class with Private and Public Members

```cpp
#include <iostream>
using namespace std;

// Define a class named Student
class Student {
private:
    int rollNumber;      // Private variable
    string name;         // Private variable

public:
    // Constructor
    Student(int r, string n) {
        rollNumber = r;
        name = n;
    }

    // Public method to display student details
    void display() {
        cout << "Roll Number: " << rollNumber << endl;
        cout << "Name: " << name << endl;
    }
};

int main() {
    // Create an object of Student
    Student s1(101, "Alice");
    s1.display(); // Call the public method

    return 0;
}
```

In this example:

- The `Student` class has private variables `rollNumber` and `name`.
- The constructor initializes these variables.
- The `display()` method is public and prints the details.
- An object `s1` is created and its details are displayed using the method.

## Allocation of Objects: Static and Dynamic

In C++, objects can be allocated either statically or dynamically, depending on how and where they are created.

### Static Allocation

Static allocation means creating objects with automatic storage duration, typically on the stack. These objects are created when the scope is entered and destroyed when the scope is exited.

```cpp
Student s1(102, "Bob"); // Static allocation
```

- Memory is managed automatically.
- Object lifetime is limited to the scope in which it is defined.

### Dynamic Allocation

Dynamic allocation means creating objects on the heap using the `new` keyword. These objects persist until they are explicitly deleted.

```cpp
Student* s2 = new Student(103, "Charlie"); // Dynamic allocation
s2->display();
delete s2; // Free the memory
```

- Memory must be managed manually using `delete`.
- Object lifetime can be controlled beyond the scope.

**Summary:**

- Use static allocation for objects with a limited lifetime and when memory management is not a concern.
- Use dynamic allocation for objects that need to persist or be shared across scopes, but remember to free the memory to avoid leaks.

## Difference Between `struct` and `class` in C++

In C++, both `struct` and `class` are used to create user-defined types that can contain data members and member functions. However, there are key differences:

| Aspect           | `struct`                           | `class`                            |
| ---------------- | ---------------------------------- | ---------------------------------- |
| Default Access   | Members are `public` by default    | Members are `private` by default   |
| Inheritance      | Base struct is `public` by default | Base class is `private` by default |
| Usage            | Traditionally for data grouping    | Used for encapsulation and OOP     |
| Function Members | Allowed                            | Allowed                            |

**Example:**

```cpp
struct Point {
    int x; // public by default
    int y;
};

class Student {
    int rollNumber; // private by default
    string name;
public:
    void setData(int r, string n) {
        rollNumber = r;
        name = n;
    }
};
```

**Summary:**  
Use `struct` for simple data structures with public members. Use `class` when you need encapsulation, data hiding, and other OOP features.

## Can `struct` Contain Member Functions in C++?

Yes, in C++, a `struct` can contain member functions just like a `class`. The main difference is that members of a `struct` are `public` by default, while members of a `class` are `private` by default.

**Example:**

```cpp
struct Rectangle {
    int width;
    int height;

    // Member function to calculate area
    int area() {
        return width * height;
    }
};

int main() {
    Rectangle rect;
    rect.width = 5;
    rect.height = 10;
    cout << "Area: " << rect.area() << endl;
    return 0;
}
```

This example shows a `struct` with a member function `area()` that calculates the area of a rectangle.

## Can `struct` Contain Member Functions in C?

No, in C, a `struct` cannot contain member functions. C structures are only used to group related data together. Functions must be defined separately and can operate on struct variables by passing them as arguments.

**Example:**

```c
#include <stdio.h>

struct Rectangle {
    int width;
    int height;
};

// Function to calculate area
int area(struct Rectangle rect) {
    return rect.width * rect.height;
}

int main() {
    struct Rectangle rect;
    rect.width = 5;
    rect.height = 10;
    printf("Area: %d\n", area(rect));
    return 0;
}
```

In this example, the function `area` is defined outside the struct and takes a struct variable as a parameter.
