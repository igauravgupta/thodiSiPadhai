## What is Copying in C++?

When you create a new object from an existing object (via copy constructor or assignment operator), the compiler decides how to copy the object.

Default behavior: Compiler provides a member-wise copy (shallow copy).

But if your class uses pointers or dynamic memory, the default shallow copy often causes problems → so we need deep copy.

---

## Non-Pointer Members (e.g., int, float, double, bool, char, std::string)

**Shallow Copy:**
Value is copied normally. Each object has its own independent copy.
✅ No issue here.

**Deep Copy:**
Same as shallow. Value is copied.
✅ No difference, always safe.

---

## Pointer Members (e.g., int*, CustomClass*)

**Shallow Copy:**
Only the pointer address is copied. Both objects point to the same memory.
❌ Modifying one object affects the other.
❌ Double free risk when destructors run.

**Deep Copy:**
New memory is allocated. Data from the original pointer is duplicated.
✅ Each object owns its own resource.
✅ Safe from side effects and double free.

---

## Nested Objects

A nested object means a class inside another class, like:

```cpp
class Inner {
public:
    int *data;
    Inner(int val) { data = new int(val); }
    ~Inner() { delete data; }
};

class Outer {
public:
    Inner innerObj;
    Outer(int val) : innerObj(val) {}
};
```

**Shallow Copy Behavior (Default):**
When Outer is copied, compiler-generated copy constructor calls Inner’s copy constructor. If Inner has no custom deep copy, its pointer is shallow-copied.
Effect: Both Outer objects share the same Inner.data.
❌ Modifying one Outer’s innerObj.data changes the other’s.
❌ Double deletion occurs when both Outers are destroyed.

**Deep Copy Behavior:**
If Inner has its own deep copy constructor, a new block of memory is allocated for innerObj.data.
Effect: Each Outer object has its own independent Inner.data.
✅ Modifications in one Outer don’t affect the other.
✅ Safe destruction, no double delete.

---

## Nested Objects (Composition, Not Pointer)

Example:

```cpp
class Outer {
    Inner innerObj; // direct object (not pointer)
};
```

The compiler will call Inner’s copy constructor when copying Outer.
If Inner has no custom copy constructor → shallow copy happens inside Inner.
If Inner implements deep copy → deep copy happens inside Inner.

👉 So yes, the behavior of nested objects depends entirely on how Inner defines its copy constructor.

---

## Nested Object Stored as a Pointer

```cpp
class Outer {
public:
    Inner* innerPtr;  // pointer to Inner
    Outer(int x) { innerPtr = new Inner(x); }
    // default shallow copy
    // Outer(const Outer& o) { innerPtr = o.innerPtr; }
};
```

**With Shallow Copy:**
Only the pointer innerPtr is copied. Both Outer objects point to the same Inner object.
❌ If you modify Outer2.innerPtr->data, it changes Outer1’s data too.
❌ Double deletion problem when both destructors call delete innerPtr;.

**With Deep Copy:**
You must write a custom copy constructor for Outer:

```cpp
Outer(const Outer& o) {
    innerPtr = new Inner(*o.innerPtr); // calls Inner’s copy constructor
}
```

Now Outer2 gets its own separate Inner object.
✅ No shared memory
✅ Safe destruction
✅ No unintended side effects

## Quick Comparison Table

| Member Type  | Shallow Copy (Default)                               | Deep Copy (Custom)                            |
| ------------ | ---------------------------------------------------- | --------------------------------------------- |
| Non-pointers | Copied normally (independent)                        | Same as shallow (independent)                 |
| Pointers     | Address copied (shared)                              | New memory allocated                          |
| Nested objs  | Uses nested’s copy ctor: if shallow → shared pointer | Calls nested’s deep copy ctor → separate copy |

---

## Rule of Thumb for Nested Objects

If a class contains another class, the behavior of copying depends on how the nested class handles its own copy constructor:

- If nested class uses shallow copy → Outer object also indirectly suffers from shallow copy problems.
- If nested class defines deep copy → Outer automatically benefits and becomes safe.

✅ In short:
Shallow copy = copies pointer addresses → nested objects with pointers are shared.
Deep copy = allocates new memory → nested objects with pointers are independent.

---

## Summary

✔ In shallow copy, all non-pointer members are copied safely by value.
✔ For pointer members, only the address is copied (danger).
✔ For nested objects declared normally (not pointer), the copy behavior depends on how the nested class (Inner) implements its own copy constructor.
✔ If you declare `Inner* innerPtr` (pointer to Inner) in Outer, then:
Default = shallow copy (just pointer copied).
To make it safe, Outer must itself implement a deep copy constructor that creates a new Inner object (`new Inner(*o.innerPtr)`).

## Rule of Three

If your class manages resources (like dynamic memory, file handles, etc.), you should explicitly define these three special member functions:

1. **Destructor** (`~ClassName()`)
2. **Copy Constructor** (`ClassName(const ClassName&)`)
3. **Copy Assignment Operator** (`ClassName& operator=(const ClassName&)`)

If you need to define any one of these, you likely need to define all three to avoid resource management bugs.

---

## Rule of Five

With C++11 and later, two more special functions are important for resource management:

4. **Move Constructor** (`ClassName(ClassName&&)`)
5. **Move Assignment Operator** (`ClassName& operator=(ClassName&&)`)

If your class defines any of the five (destructor, copy/move constructor, copy/move assignment), you should consider defining all five for safe and efficient resource handling.

---

## Rule of Three vs Rule of Five Comparison Table

| Rule          | Applies To      | Functions to Define                                                                                |
| ------------- | --------------- | -------------------------------------------------------------------------------------------------- |
| Rule of Three | Pre-C++11       | Destructor, Copy Constructor, Copy Assignment Operator                                             |
| Rule of Five  | C++11 and later | Destructor, Copy Constructor, Copy Assignment Operator, Move Constructor, Move Assignment Operator |
