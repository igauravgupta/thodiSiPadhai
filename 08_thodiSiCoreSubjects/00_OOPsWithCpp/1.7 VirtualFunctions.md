# Virtual Functions in C++

## What is a Virtual Function?

A virtual function is a member function in a base class that you expect to be overridden in a derived class.

It tells the compiler:
👉 “Don’t bind this function call at compile time (early binding). Instead, wait until runtime to decide which version of the function to call.”

---

## Why Do We Need Virtual Functions?

Without virtual, function calls through a base class pointer/reference always use early binding (base class version).
But in polymorphism, we want the derived class’s overridden function to run even when accessed through a base class pointer/reference.

✅ Use Case: When working with inheritance and polymorphism

- Base class defines a common interface.
- Derived classes provide their own implementations.
- Base class pointer/reference can point to any derived class object, and the correct method executes.

The method that gets called is determined at runtime based on the actual object that the base class pointer refers to.
The base class in this context is known as the virtual base class.

---

## How to Write a Virtual Function

We declare a function with the keyword `virtual` in the base class.

---

### Example: Without Virtual Function (Early Binding)

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    void show() { cout << "Base show()\n"; }
};

class Derived : public Base {
public:
    void show() { cout << "Derived show()\n"; }
};

int main() {
    Base* ptr;
    Derived d;
    ptr = &d;
    ptr->show();  // Output: Base show() ❌ (not polymorphic)
}
```

Here, even though `ptr` points to a `Derived` object, it calls `Base::show()` because binding is done at compile time.

---

### Example: With Virtual Function (Late Binding)

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void show() { // virtual keyword
        cout << "Base show()\n";
    }
};

class Derived : public Base {
public:
    void show() override { // optional "override" keyword (C++11+)
        cout << "Derived show()\n";
    }
};

int main() {
    Base* ptr;
    Derived d;
    ptr = &d;
    ptr->show();  // Output: Derived show() ✅ (runtime polymorphism)
}
```

---

## Key Points about Virtual Functions

- Declared in base class with `virtual`.
- Overridden in derived class (same function signature).
- When accessed through base pointer/reference, the derived class version executes.
- Implemented using V-Table (Virtual Table) mechanism.
- If derived class does not override → base class version is used.
- If a class has virtual functions, its destructor should usually also be virtual (to avoid memory leaks in polymorphic deletion).

---

## Summary

Virtual Function = Function in base class meant to be overridden in derived class.

Why? → To achieve runtime polymorphism.

How? → Use the `virtual` keyword in base class, override in derived class.

---

## Pure Virtual Functions and Abstract Classes

### What is a Pure Virtual Function?

A pure virtual function is a virtual function with no implementation in the base class. It is declared by assigning `= 0` in its declaration.

```cpp
class Base {
public:
    virtual void show() = 0; // Pure virtual function
};
```

### Abstract Class

A class containing at least one pure virtual function is called an **abstract class**. You cannot create objects of an abstract class directly.

Abstract classes are used to define interfaces for derived classes.

---

### Example: Pure Virtual Function and Abstract Class

```cpp
#include <iostream>
using namespace std;

class Shape {
public:
    virtual void draw() = 0; // Pure virtual function
};

class Circle : public Shape {
public:
    void draw() override {
        cout << "Drawing Circle\n";
    }
};

int main() {
    // Shape s; // ❌ Error: Cannot instantiate abstract class
    Circle c;
    Shape* ptr = &c;
    ptr->draw(); // Output: Drawing Circle
}
```

---

**Key Points:**

- Pure virtual functions force derived classes to provide their own implementation.
- Abstract classes cannot be instantiated.
- Used to define common interfaces for derived classes.
- Enables strong runtime polymorphism and design flexibility.
