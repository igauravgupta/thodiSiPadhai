# Need for DBMS

- A Database Management System or DBMS is a software that runs allows proper storing, organizing and managing large amounts of data. It ensures data consistency, integrity and security while allowing multiple users to access and manipulate data simultaneously.

# Limitations of Traditional File Systems

- **Data Redundancy and Inconsistency**: Storing data in multiple files often leads to duplication and inconsistency, making it difficult to maintain accurate information.
- **Difficulty in Accessing Data**: Retrieving specific data from large sets of files requires complex programming and is time-consuming.
- **Limited Data Sharing**: File systems do not support concurrent access by multiple users, leading to issues in data sharing and collaboration.
- **Poor Data Security**: Traditional file systems offer limited mechanisms for controlling access and protecting sensitive data.
- **Lack of Data Integrity**: Ensuring data correctness and enforcing constraints is challenging without a centralized management system.
- **No Support for Transactions**: File systems cannot guarantee atomicity, consistency, isolation, and durability (ACID properties) for operations.
- **Scalability Issues**: Managing and organizing large volumes of data becomes inefficient as the data grows.
