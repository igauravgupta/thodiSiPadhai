## What are Integrity Constraints?

Integrity constraints are rules that ensure the accuracy, consistency, and reliability of data in a database. They help enforce business rules and prevent invalid data from being stored.

---

## 1. Domain Constraints

Domain constraints restrict the values of an attribute to a specific set (domain).

### Types of Domain Constraints

- **NOT NULL Constraint:** Ensures that a column cannot have NULL values.
- **CHECK Constraint:** Ensures that values in a column satisfy a specific condition.

**Example:**

```sql
CREATE TABLE Student (
    Age INT CHECK (Age >= 0 AND Age <= 120) NOT NULL
);
```

**Why Important:**  
Domain constraints prevent invalid or inconsistent data, making the database reliable and easier to maintain.

---

## 2. Entity Integrity Constraints

Entity integrity ensures that each row in a table is uniquely identified, typically using a primary key.

### Types of Entity Integrity Constraints

- **PRIMARY KEY Constraint:** Uniquely identifies each row and cannot be NULL.

**Example:**

```sql
CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,
    Name VARCHAR(100)
);
```

**Why Important:**  
Guarantees that every record is unique and accessible.

---

## 3. Key Constraints

Key constraints ensure that certain attributes (keys) uniquely identify a tuple (row) in a relation.

### Types of Key Constraints

- **PRIMARY KEY:** Uniquely identifies each row.
- **UNIQUE KEY:** Ensures all values in a column are unique.
- **CANDIDATE KEY:** Any column or set of columns that can qualify as a primary key.

**Example:**

```sql
CREATE TABLE Product (
    ProductID INT PRIMARY KEY,
    SerialNumber VARCHAR(50) UNIQUE
);
```

**Why Important:**  
Prevents duplicate records and maintains data integrity.

---

## 4. Referential Integrity Constraints

Referential integrity ensures that a foreign key value always refers to an existing, valid record in another table.

### Types of Referential Integrity Constraints

- **FOREIGN KEY Constraint:** Links two tables together, referencing a primary or unique key in another table.
- **ON DELETE / ON UPDATE Actions:** Control what happens when referenced data is changed or deleted (CASCADE, SET NULL, RESTRICT).

**Example:**

```sql
CREATE TABLE Orders (
    OrderID INT PRIMARY KEY,
    CustomerID INT,
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);
```

**Why Important:**  
Maintains relationships between tables and prevents orphaned records.

---

## 5. Assertions

Assertions are conditions specified on the database that must always be true. They enforce complex business rules across tables.

**Example:**

```sql
CREATE ASSERTION SalaryCheck
CHECK (NOT EXISTS (
    SELECT * FROM Employee WHERE Salary < 0
));
```

**Why Important:**  
Enforces rules that cannot be handled by simple constraints.

---

## 6. Triggers

Triggers are procedural code that is automatically executed in response to certain events (INSERT, UPDATE, DELETE) on a table.

### Types of Triggers

- **BEFORE Trigger:** Executes before the event.
- **AFTER Trigger:** Executes after the event.
- **INSTEAD OF Trigger:** Executes instead of the event (mainly in views).

**Example:**

```sql
CREATE TRIGGER UpdateTimestamp
AFTER UPDATE ON Orders
FOR EACH ROW
BEGIN
    UPDATE Orders SET LastModified = CURRENT_TIMESTAMP WHERE OrderID = NEW.OrderID;
END;
```

**Why Important:**  
Automates actions, enforces rules, and maintains audit trails.
